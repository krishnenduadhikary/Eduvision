Pointers in C

Unlocking direct memory manipulation and advanced programming techniques.

1. Introduction to Pointers

A pointerA derived data type in C. is a variable that stores/holds the memory address of another variable of the same data type. It is also known as a locator or indicator that points to an address of a value.

Here, the 'pointer' variable stores the address where 'variable' (which holds the actual value) is located in memory.

Benefits of Using Pointers:

Efficient Array & Structure Handling

Pointers provide an efficient way to access and manipulate elements of arrays and members of structures.

Function Arguments

Pointers allow passing arguments to functions by reference, enabling functions to modify the original variables. They also allow passing functions as arguments to other functions.

Reduced Program Length & Execution Time

Direct memory access can sometimes lead to more concise code and faster execution, especially in system-level programming.

Dynamic Memory Management

Pointers are essential for dynamic memory allocation (e.g., using `malloc`, `calloc`), allowing programs to request memory at runtime.

2. Pointer Basics: Declaration, Initialization, and Operators

A pointer is declared by specifying the data type of the variable it will point to, followed by an asterisk (`*`) and the pointer variable name.

Syntax:

data_type *pointer_variable_name;

Examples:

int *p; // p is a pointer to an integer char *cptr; // cptr is a pointer to a character float *fptr; // fptr is a pointer to a float

Note: A `void` pointer (`void *ptr;`) is a generic pointer that can point to any data type. However, it cannot be dereferenced directly and must be cast to a specific pointer type before accessing the value it points to. It's often used in functions like `malloc` which return a generic memory block address.

Pointer initialization is the process of assigning the address of a variable to a pointer variable. A pointer variable must contain the address of a variable of the same data type it is declared to point to.

Syntax:

data_type variable_name; data_type *pointer_name; pointer_name = &variable_name; // Assigning address of 'variable_name' to 'pointer_name'

Example:


int a = 10;
int *ptr;    // Pointer declaration
ptr = &a;    // Pointer initialization (ptr now holds the address of a)

// Declaration and initialization together
int b = 20;
int *ptr_b = &b;

Important: A pointer variable always points to the same type of data. 


float x;
int *p_int;
// p_int = &x; // ERROR: Type mismatch (int pointer cannot store address of float)

Pointer Example Visual:

In the figure, pointer `p` (located at address `aaa3`) stores the address `fff4`, which is the location of the `number` variable. The value of `number` is 50.

C provides two fundamental pointer operators:

Symbol Name Description
& Address-of Operator (Reference Operator) Returns the memory address of its operand (a variable).
* Indirection Operator (Dereference Operator) Accesses the value stored at the memory address pointed to by its operand (a pointer).

Dereferencing a Pointer:

Once a pointer has been assigned the address of a variable, it can be dereferencedAccessing the value stored at the memory location pointed to by the pointer. using the indirection operator (`*`) to access the value of that variable.


#include 

int main() {
    int a = 10;
    int *p;      // Declare an integer pointer

    p = &a;      // Assign the address of 'a' to 'p'

    printf("Value of a: %d\n", a);
    printf("Value at address stored in p (*p): %d\n", *p); // Dereferencing p

    printf("Address of a (&a): %p\n", (void *)&a); // Use %p for addresses, cast to void*
    printf("Address stored in p (value of p): %p\n", (void *)p);
    printf("Address of pointer p itself (&p): %p\n", (void *)&p);

    *p = 25; // Modifying the value of 'a' through the pointer 'p'
    printf("New value of a (after *p = 25): %d\n", a);

    return 0;
}
/*
Example Output (addresses will vary):
Value of a: 10
Value at address stored in p (*p): 10
Address of a (&a): 0x7ffc12345678
Address stored in p (value of p): 0x7ffc12345678
Address of pointer p itself (&p): 0x7ffc1234567c
New value of a (after *p = 25): 25
*/

Note: `%p` is the correct format specifier for printing pointer addresses with `printf`. It's good practice to cast the pointer to `(void *)` when using `%p`.

3. Key Points to Remember About Pointers

  • A normal variable stores a value, whereas a pointer variable stores the address of another variable.
  • The content (value) of a C pointer is always a whole number representing a memory address.
  • It's good practice to initialize pointers, often to `NULL` if they don't point to a valid address immediately. Example: `int *p = NULL;`
  • The value of a NULL pointerA pointer that does not point to any valid memory location. Often represented by the macro `NULL` (typically `(void*)0`). is conceptually zero. Dereferencing a NULL pointer leads to undefined behavior (often a crash).
  • The `&` symbol (address-of operator) is used to get the address of a variable.
  • The `*` symbol (indirection/dereference operator) is used to get the value stored at the address pointed to by a pointer.
  • If a pointer is assigned `NULL`, it means it is pointing to nothing.
  • Two pointers (of the same type, typically pointing within the same array) can be subtracted to find the number of elements between them.
  • Pointer addition (pointer + pointer), multiplication, and division are generally not allowed or meaningful. However, adding/subtracting an integer to/from a pointer (pointer arithmetic) is valid and common.
  • The size of any pointer variable is typically the same on a given system architecture (e.g., 4 bytes on a 32-bit system, 8 bytes on a 64-bit system), regardless of the data type it points to. (The slide saying "2 byte for 16 bit compiler" is historically relevant but modern systems are different).

4. Pointer Arithmetic

C allows certain arithmetic operations on pointers. These operations are scaled according to the size of the data type the pointer points to.

Four arithmetic operations can be used on pointers:

  • Increment (`++`)
  • Decrement (`--`)
  • Addition of an integer (`+`)
  • Subtraction of an integer (`-`)
  • Subtraction of two pointers (of the same type, yields an integer representing the number of elements between them)

Incrementing a Pointer:

When a pointer is incremented, its value (the address it holds) increases by the size of the data type it points to.
Formula: `new_address = current_address + (1 * sizeof(data_type))`

Pictorial Representation of Pointer Increment:

Data Type of Pointer Older Address Stored Next Address after Increment (ptr++)
`int` (assuming 4 bytes) 1000 1004
`float` (assuming 4 bytes) 1000 1004
`char` (assuming 1 byte) 1000 1001
`int` (assuming 2 bytes - older systems) 1000 1002 

#include 
int main() {
    int number = 50;
    int *p = &number;
    printf("Address of p variable is %p\n", (void *)p);
    p++; // Increment pointer
    printf("After increment: Address of p variable is %p\n", (void *)p);
    // Output difference will be sizeof(int)
    return 0;
}

Decrementing a Pointer:

When a pointer is decremented, its value decreases by the size of the data type it points to.
Formula: `new_address = current_address - (1 * sizeof(data_type))` 


#include 
int main() {
    int arr[] = {10, 20, 30};
    int *p = &arr[1]; // p points to 20
    printf("Address stored in p: %p\n", (void *)p);
    p--; // Decrement pointer, now points to 10
    printf("After decrement, address in p: %p, value *p: %d\n", (void *)p, *p);
    return 0;
}

Adding an Integer to a Pointer:

Adding an integer `n` to a pointer moves the pointer forward by `n * sizeof(data_type)` bytes.
Formula: `new_address = current_address + (integer_value * sizeof(data_type))` 


#include 
int main() {
    int numbers[] = {10, 20, 30, 40, 50};
    int *ptr = numbers; // ptr points to numbers[0]
    printf("Initial address: %p, value: %d\n", (void *)ptr, *ptr);
    ptr = ptr + 3; // Move pointer 3 elements forward
    printf("After ptr = ptr + 3, address: %p, value: %d\n", (void *)ptr, *ptr); // Points to 40
    return 0;
}

Subtracting an Integer from a Pointer:

Subtracting an integer `n` from a pointer moves the pointer backward by `n * sizeof(data_type)` bytes.
Formula: `new_address = current_address - (integer_value * sizeof(data_type))`

5. Pointers and Arrays/Strings

Pointers and arrays have a close relationship in C. The name of an array, when used in most expressions, decays into a pointer to its first element.

When an array is declared, the compiler allocates a contiguous block of memory to hold all its elements. The base address (address of the first element) is associated with the array name.

The array name `arr` itself acts like a constant pointer to `arr[0]`.
Therefore, `arr` is equivalent to `&arr[0]`.

We can declare a pointer of the array's element type and make it point to the array:


int arr[5] = {1, 2, 3, 4, 5};
int *p;
p = arr;      // p now points to arr[0]. Same as p = &arr[0];

Now, we can access array elements using the pointer `p` and pointer arithmetic:

  • `*p` is equivalent to `arr[0]`
  • `*(p + 1)` is equivalent to `arr[1]`
  • `*(p + i)` is equivalent to `arr[i]`

Similarly:

  • `p + i` is equivalent to `&arr[i]`

#include 
int main() {
    int arr[5] = {10, 20, 30, 40, 50};
    int *p = arr; // p points to arr[0]
    int i;

    printf("Accessing array elements using pointer:\n");
    for (i = 0; i < 5; i++) {
        printf("Value at *(p + %d): %d (or p[%d])\n", i, *(p + i), i);
        // printf("Address (p + %d): %p\n", i, (void*)(p + i));
    }

    printf("\nAccessing array elements using array name as pointer:\n");
     for (i = 0; i < 5; i++) {
        printf("Value at *(arr + %d): %d\n", i, *(arr + i));
    }
    return 0;
}

Important Note: While an array name often decays to a pointer, it is a constant pointer. You cannot change the address that the array name itself points to.
arr = some_other_address; // Compile-time error
arr++; // Compile-time error
However, a pointer variable like `p` that points to an array can be changed: `p++;` is valid.

Since strings are character arrays, pointers can be used to manipulate strings effectively. A string literal itself (e.g., `"Hello"`) often evaluates to a pointer to its first character in memory (usually read-only memory).


#include 
#include  // For strlen

int main() {
    char str[] = "Hello World"; // str is a char array
    char *ptr_str = "Welcome";  // ptr_str is a pointer to a string literal

    printf("String using array: %s\n", str);
    printf("String using pointer: %s\n", ptr_str);

    // Traversing str using a pointer
    char *p = str;
    printf("Traversing '%s' with a pointer:\n", str);
    while (*p != '\0') {
        printf("%c", *p);
        p++;
    }
    printf("\n");

    // Modifying array-based string is okay
    str[0] = 'J';
    printf("Modified array string: %s\n", str); // Jello World

    // Attempting to modify string literal via ptr_str leads to undefined behavior
    // ptr_str[0] = 'B'; // DANGEROUS - DO NOT DO THIS for string literals

    return 0;
}

When an array is passed to a function, what is actually passed is the address of its first element (i.e., a pointer to the first element). This means the function operates on the original array, not a copy (similar to call by reference).

You can declare the function parameter in three equivalent ways:

  1. As a pointer: `void myFunction(int *param)`
  2. As a sized array: `void myFunction(int param[10])` (The size `10` is often ignored by the compiler; it still just gets a pointer. It's more for documentation.)
  3. As an unsized array: `void myFunction(int param[])`

All three forms effectively treat `param` as `int *` within the function.

Example (Calculating average of array elements):


#include 

// Function declaration (prototype)
double getAverage(int arr[], int size); // Or int *arr

int main() {
    int balance[5] = {1000, 2, 3, 17, 50};
    double avg;

    avg = getAverage(balance, 5); // Pass array (its address) and its size
    printf("Average value is: %f\n", avg); // Output: Average value is: 214.400000

    return 0;
}

// Function definition
double getAverage(int arr[], int size) { // arr is effectively a pointer here
    int i;
    double avg_val;
    double sum = 0;

    for (i = 0; i < size; ++i) {
        sum += arr[i]; // Can use array notation arr[i]
        // or pointer notation *(arr + i)
    }
    avg_val = sum / size;
    return avg_val;
}

6. Advanced Pointer Concepts

A pointer can point to another pointer. This is known as a pointer to a pointer, or a double pointer. It stores the address of another pointer variable.

Here, `p2` (at address `fff0`) stores the address of pointer `p` (which is `fff2`). Pointer `p` (at address `fff2`) stores the address of variable `number` (which is `fff4`). Variable `number` (at address `fff4`) holds the value `50`.

Syntax:

data_type **pointer_to_pointer_name;

Example:


#include 
// #include  // For clrscr, getch

int main() {
    int number = 50;
    int *p;    // Pointer to int
    int **p2;  // Pointer to pointer to int

    // clrscr();
    p = &number;  // p stores address of number
    p2 = &p;      // p2 stores address of pointer p

    printf("Address of number variable (&number): %p\n", (void *)&number);
    printf("Address stored in p variable (value of p): %p\n", (void *)p);
    printf("Value pointed to by *p: %d\n", *p);

    printf("Address of p variable (&p): %p\n", (void *)&p);
    printf("Address stored in p2 variable (value of p2): %p\n", (void *)p2);
    printf("Value pointed to by *p2 (address stored in p): %p\n", (void *)*p2);
    printf("Value pointed to by **p2 (value of number): %d\n", **p2);

    // getch();
    return 0;
}
/*
Example Output (addresses will vary):
Address of number variable (&number): 0x7ffee1c5a87c
Address stored in p variable (value of p): 0x7ffee1c5a87c
Value pointed to by *p: 50
Address of p variable (&p): 0x7ffee1c5a870
Address stored in p2 variable (value of p2): 0x7ffee1c5a870
Value pointed to by *p2 (address stored in p): 0x7ffee1c5a87c
Value pointed to by **p2 (value of number): 50
*/

Double pointers are often used in functions that need to modify a pointer passed as an argument (e.g., in dynamic memory allocation scenarios or when working with arrays of strings where each string is a pointer).

Just as pointers can point to data variables, they can also point to functions. A function pointer stores the starting address (entry point) of the executable code of a function.

Key Points:

  • A pointer to a function points to the address of the executable code of the function.
  • We can use function pointers to call functions and to pass functions as arguments to other functions (useful for callbacks or strategy patterns).
  • Pointer arithmetic (like `++`, `--`, `+integer`) cannot be performed on function pointers.
  • The type of a pointer to a function is based on both the function's return type and the types of its parameters.
  • The declaration of a function pointer must have the pointer name in parentheses (e.g., `(*ptr_name)`). Without them, it might be interpreted as a function returning a pointer.

Syntax for Declaration:

return_type (*pointer_name)(parameter_type_list);

Example:


#include 

// A sample function
int sum(int num1, int num2) {
    return num1 + num2;
}

int main() {
    // Declare a function pointer 'f2p' that can point to a function
    // which takes two integers as arguments and returns an integer.
    int (*f2p)(int, int);

    // Assign the address of the 'sum' function to the pointer.
    // The function name itself (without parentheses) gives its address.
    f2p = sum; // or f2p = ∑ (both are generally acceptable)

    int op1, op2;

    // Call the function 'sum' using the function pointer 'f2p'
    op1 = f2p(10, 13);

    // Call the function 'sum' directly
    op2 = sum(10, 13);

    printf("Output 1 - for function call via Pointer: %d\n", op1);
    printf("Output 2 - for direct function call: %d\n", op2);

    // You would have noticed that the output of both statements is same
    // f2p(10, 13) == sum(10, 13)
    // which means in generic sense you can write it out as:
    // pointer_name(argument list) == function(same argument list)

    return 0;
}
/*
Output:
Output 1 - for function call via Pointer: 23
Output 2 - for direct function call: 23
*/

Declaration example: `double (*p2f)(double, char);` declares `p2f` as a pointer to a function that takes a `double` and a `char` as arguments and returns a `double`.

A `void` pointer is a special type of pointer that has no associated data type. It can hold the address of any type of variable. It's also known as a general-purpose pointer.

Syntax:

void *pointer_name;

Key Points:

  • It can store the address of any type of variable.
  • It does not have any data type associated with it.
  • A `void` pointer cannot be dereferenced directly. The compiler doesn't know the size or type of data it points to. You must explicitly cast it to an appropriate pointer type before dereferencing.
  • Pointer arithmetic is not directly possible on `void` pointers (because the size of the pointed-to type is unknown). Some compilers might allow it by assuming `sizeof(void)` is 1, but this is non-standard and should be avoided.

Advantages:

  1. Functions like `malloc()` and `calloc()` return `void *`, allowing them to allocate memory for any data type. The returned pointer is then cast to the desired type.
  2. `void` pointers are used to implement generic functions in C (functions that can operate on data of different types).

Example:


#include 

int main() {
    int inum = 10;
    char cnum = 'A';
    float fnum = 3.14f;

    void *ptr; // ptr is declared as void pointer

    ptr = &inum;
    printf("Value of inum (via void pointer): %d\n", *( (int*)ptr ) ); // Cast to int* before dereferencing

    ptr = &cnum;
    printf("Value of cnum (via void pointer): %c\n", *( (char*)ptr ) );

    ptr = &fnum;
    printf("Value of fnum (via void pointer): %f\n", *( (float*)ptr ) );

    // Direct dereferencing error:
    // printf("%d", *ptr); // COMPILE ERROR: 'void*' is not a pointer-to-object type

    return 0;
}

An array of pointers is an array where each element is a pointer variable. Each pointer in the array can point to a memory location of its declared base type.

Syntax:

data_type *array_name[size];

Example:


#include 

const int MAX = 3; // Using const int for array size

int main () {
   int  var[] = {10, 100, 200}; // An array of integers
   int *ptr[MAX];               // An array of MAX integer pointers
   int i;

   for (i = 0; i < MAX; i++) {
      ptr[i] = &var[i]; /* assign the address of integer. */
   }
   
   for (i = 0; i < MAX; i++) {
      printf("Value of var[%d] = %d (Address: %p, Stored in ptr[%d]: %p)\n", 
             i, *ptr[i], (void*)&var[i], i, (void*)ptr[i]);
   }
   
   return 0;
}
/*
Output:
Value of var[0] = 10 (Address: 0x..., Stored in ptr[0]: 0x...)
Value of var[1] = 100 (Address: 0x..., Stored in ptr[1]: 0x...)
Value of var[2] = 200 (Address: 0x..., Stored in ptr[2]: 0x...)
(Addresses will vary)
*/

Arrays of pointers are commonly used for storing an array of strings (where each string is itself a character array, and the array of pointers stores the base address of each string).

7. Dynamic Memory Allocation

Dynamic memory allocation is the process of allocating memory at runtime, rather than at compile time. This allows programs to request memory as needed and release it when no longer required. C provides several library functions for dynamic memory allocation, primarily found in ``.

Static vs. Dynamic Memory Allocation:

Static Memory Allocation Dynamic Memory Allocation
Memory is allocated at compile time. Memory is allocated at runtime.
Memory size cannot be changed while the program is executing. Memory size can be changed (e.g., using `realloc`) while the program is executing.
Typically used for arrays with fixed sizes, global/static variables. Memory is allocated on the stack (for local variables) or data segment. Typically used for linked lists, trees, or when the amount of memory needed is unknown until runtime. Memory is allocated from the HeapA region of memory available for dynamic allocation..

Dynamic Memory Allocation Functions:

  • `malloc()` (memory allocation): Allocates a single block of requested memory. It does not initialize the memory (contains garbage values). Returns a `void*` pointer to the allocated block, or `NULL` if allocation fails.
    ptr = (cast_type*) malloc(byte_size);
  • `calloc()` (contiguous allocation): Allocates memory for an array of elements, initializes all bytes in the allocated block to zero. Returns a `void*` pointer, or `NULL` on failure.
    ptr = (cast_type*) calloc(number_of_elements, size_of_each_element_in_bytes);
  • `realloc()` (re-allocation): Changes the size of a previously allocated memory block (pointed to by `ptr`) to a new size. It may move the memory block to a new location. Returns `void*` to the (potentially new) block, or `NULL` on failure.
    ptr = (cast_type*) realloc(existing_ptr, new_byte_size);
  • `free()`: Deallocates (releases) a previously allocated memory block, making it available for future use. It's crucial to free dynamically allocated memory to prevent memory leaksWhen a program allocates memory but loses all references to it, making it unusable but still occupied..
    free(ptr);

#include 
#include  // For malloc, free, exit

int main() {
    int n, i, *ptr, sum = 0;

    printf("Enter number of elements: ");
    scanf("%d", &n);

    // Dynamically allocate memory using malloc()
    ptr = (int*) malloc(n * sizeof(int));

    // Check if the memory has been successfully allocated by malloc or not
    if (ptr == NULL) {
        printf("Error! Memory not allocated.\n");
        exit(0); // Terminate program
    }

    printf("Enter elements of array:\n");
    for (i = 0; i < n; ++i) {
        printf("Enter element %d: ", i + 1);
        scanf("%d", ptr + i); // or &ptr[i]
        sum += *(ptr + i);    // or ptr[i]
    }

    printf("Sum = %d\n", sum);

    // Deallocate the memory
    free(ptr);
    ptr = NULL; // Good practice to set pointer to NULL after freeing

    return 0;
}
                    

#include 
#include 

int main() {
    int n, i, *ptr, sum = 0;

    printf("Enter number of elements: ");
    scanf("%d", &n);

    // Dynamically allocate memory using calloc()
    // It allocates 'n' blocks, each of size 'sizeof(int)'
    // and initializes all to zero.
    ptr = (int*) calloc(n, sizeof(int));

    if (ptr == NULL) {
        printf("Error! Memory not allocated.\n");
        exit(0);
    }

    printf("Memory successfully allocated using calloc.\n");
    // Since calloc initializes to 0, we can demonstrate that
    printf("Initial elements (should be 0):\n");
    for (i = 0; i < n; ++i) {
        printf("%d ", *(ptr + i));
    }
    printf("\n");

    printf("Enter elements of array:\n");
    for (i = 0; i < n; ++i) {
        printf("Enter element %d: ", i + 1);
        scanf("%d", ptr + i);
        sum += *(ptr + i);
    }

    printf("Sum = %d\n", sum);
    free(ptr);
    ptr = NULL;
    return 0;
}

`calloc()` `malloc()`

Number of arguments is 2: `calloc(num_elements, element_size)` Number of arguments is 1: `malloc(total_byte_size)`
Syntax: `(cast_type*)calloc(blocks, size_of_block);` Syntax: `(cast_type*)malloc(Size_in_bytes);`

Memory Allocation Process (Heap): Global variables, static variables, and program instructions get memory in permanent storage areas. Local variables are typically stored on the Stack. The memory space between these two regions is known as the Heap area. This region is used for dynamic memory allocation during program execution. The size of the heap can change.

8. Command-Line Arguments

It's possible to pass values from the command line to your C programs when they are executed. These values are called command-line arguments and are particularly useful for controlling program behavior from outside, instead of hardcoding values.

To support command-line arguments, the structure of the `main()` function needs to be changed:

Syntax:

int main(int argc, char *argv[]) { // ... code ... return 0; }
  • `argc` (argument count): An integer that stores the number of command-line arguments passed to the program. It includes the program name itself as the first argument. So, `argc` is always at least 1.
  • `argv` (argument vector): An array of character pointers (array of strings). Each element `argv[i]` points to a string representing an argument.
    • `argv[0]` is the name of the program.
    • `argv[1]` is the first actual argument.
    • ...
    • `argv[argc-1]` is the last argument.
    • `argv[argc]` is a NULL pointer (by convention, marking the end of the argument list).

Example:


#include 
// #include  // For clrscr, getch - not needed for standard command line args

int main(int argc, char *argv[]) { // Use standard int main()
    int i;
    // clrscr(); // Non-standard, remove

    printf("Total number of arguments: %d\n", argc);
    printf("Program name: %s\n", argv[0]);

    if (argc > 1) {
        printf("Arguments passed:\n");
        for (i = 1; i < argc; i++) { // Start from 1 to skip program name
            printf("Argument %d: %s\n", i, argv[i]);
        }
    } else {
        printf("No additional arguments passed.\n");
    }
    // getch(); // Not typically used in command-line programs
    return 0;
}

How to Run with Command-Line Arguments:

Compile the program (e.g., `gcc mycmd.c -o mycmd`). Then run it from the terminal:

./mycmd arg1 "another argument" 123

Example Output (for the command above):

Total number of arguments: 4
Program name: ./mycmd
Arguments passed:
Argument 1: arg1
Argument 2: another argument
Argument 3: 123

Note:

  • Arguments are passed as strings. If you need to use them as numbers, you'll need to convert them (e.g., using `atoi()`, `atof()` from ``).
  • Arguments containing spaces should be enclosed in quotes (e.g., `"hello world"`).
  • Command-line argument programs are typically run from a terminal/command prompt, not directly from some IDEs' "run" button without specific configuration for arguments.
