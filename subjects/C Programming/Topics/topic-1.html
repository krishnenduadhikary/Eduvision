Introduction to C Programming

Discover the power, elegance, and enduring relevance of the C programming language.

1. History and Importance of C

The C programming language has a rich history and continues to be one of the most influential languages in the world of computing.

  • Developed in 1972The year C was officially born. by Dennis RitchieOften called the "father of C". at Bell Laboratories of AT&T (American Telephone & Telegraph), located in the U.S.A.
  • It was initially designed for programming in the UNIX operating system.C and UNIX were developed in tandem.
  • C evolved from earlier languages like ALGOLALGOrithmic Language, introduced block structure. (1960), BCPLBasic Combined Programming Language, by Martin Richards. (1967, Martin Richards), and BDeveloped by Ken Thompson, a predecessor to C. (1970, Ken Thompson). C restored some generality lost in B and BCPL.
  • Many software tools, C compilers themselves, and major parts of popular operating systems like Windows, UNIX, and Linux are still written in C.
  • It was developed to overcome problems of previous languages and provide a balance between high-level readability and low-level control.

Why is C Still Important?

  • Foundation for Other Languages: Many modern languages (C++, Java, C#, Python, JavaScript) have syntax or concepts influenced by C.
  • System Programming: Ideal for developing operating systems, compilers, drivers, and embedded systems due to its efficiency and hardware access capabilities.
  • Performance: C programs are generally very fast because they compile directly to machine code and offer fine-grained memory control.
  • Portability: C compilers are available for almost all platforms, making it relatively easy to write code that runs on different systems.

2. Key Features of C

  • General-Purpose: Can be used to write a wide variety of applications.
  • Middle-Level Language: Combines features of high-level languages (readability, structure) with the functionality of assembly language (low-level memory access, bit manipulation).
  • Structured Language: Programs can be broken down into modules or functions, promoting organized and maintainable code.
  • Rich Library: Provides a vast collection of built-in functions for various operations.
  • Efficient and Fast: C programs are known for their speed and efficient use of system resources.
  • Portable: Code written in C can often be compiled and run on different machines with minimal or no modification.
  • Memory Management: Offers direct control over memory allocation and deallocation (e.g., using `malloc()`, `calloc()`, `free()`).
  • Pointers: Allows direct memory manipulation, which is powerful but requires careful handling.
  • Modularity: C programs can be divided into smaller, manageable modules.
  • Extensible: New features can be added through libraries.

3. Structure of a C Program

A typical C program consists of several sections. While not all are mandatory for every program, understanding the general structure is key.

  1. Comment Line(s): Explanatory notes ignored by the compiler.
  2. Preprocessor Directive(s): Instructions for the C preprocessor (e.g., `#include`, `#define`).
  3. Global Declaration(s): Variables or functions declared outside any function, accessible throughout the program.
  4. `main()` Function: The entry point of every C program. Execution begins here.
    • Local Variable Declarations: Variables declared inside a function.
    • Statements: Instructions to be executed.
    • Return Statement: (Optional for `void main()`, but good practice for `int main()`).
  5. User-Defined Function(s): Functions created by the programmer to perform specific tasks.

Visual Structure Example


// 1. Comment Line
#include   // 2. Preprocessor Directive

int global_var = 100; // 3. Global Declaration

// 5. User-defined function (example)
void myFunction() {
    printf("Inside myFunction\n");
}

int main() { // 4. main() Function
    // Local Variable Declaration
    int local_var = 10;
    char message[] = "Hello, C!";

    // Statements
    printf("%s\n", message); // Output to console
    printf("Local var: %d\n", local_var);
    printf("Global var: %d\n", global_var);
    myFunction(); // Calling user-defined function

    return 0; // Return statement (indicates successful execution)
}

The main() function is where the program execution begins and ends. It can return an integer value to the operating system (typically `0` for success) if declared as `int main()`, or no value if declared as `void main()` (though `int main()` is standard).

4. Compilation and Execution Process

Transforming C source code into an executable program involves several steps, typically managed by a compilerA program that translates source code from a high-level language to a lower-level language (e.g., assembly or machine code)..

C Program Lifecycle

→

→
Compiler
→
Assembler
→
Object Code (*.obj or *.o file)
→
Linker
→
Executable Code (*.exe or a.out)
→

  • Source Code: The human-readable C code you write, typically saved with a `.c` extension.
  • Preprocessor: Processes directives (lines starting with `#`) like `#include` (includes header files) and `#define` (defines macros). It expands the source code.
  • Compiler: Translates the preprocessed code into assembly language, then into machine-readable object code (`.obj` or `.o` files). It checks for syntax errors.
  • Assembler: Converts assembly language (if generated as an intermediate step) to object code. Some compilers combine this step.
  • Linker: Combines your program's object code with object code from library functions (e.g., `printf` from `stdio.h`) and other object files to create a single executable file.
  • Loader: When you run the program, the loader loads the executable file from disk into the computer's main memory (RAM) for execution by the CPU.

5. C Tokens: Keywords, Identifiers, Constants

Tokens are the smallest individual units in a C program. C has six types of tokens:

  1. Identifiers
  2. Keywords
  3. Constants
  4. Strings
  5. Special Symbols
  6. Operators

1. C Character Set

The characters that can be used to form words, numbers, and expressions depend upon the computer on which the program is run. However, a subset of characters is available that can be used on most personal, micro, and mainframe computers.

3. Identifiers

IdentifiersNames given to entities like variables, functions, arrays, etc. refer to the names of variables, functions, arrays, structures, etc. They are user-defined names.

Rules for Naming Identifiers:
  • Must consist of only alphabets (uppercase or lowercase), digits, and underscore (`_`).
  • The first character must be an alphabet or an underscore. It cannot be a digit.
  • Cannot be a keyword.
  • C is case-sensitive (e.g., `age`, `Age`, and `AGE` are different identifiers).
  • Should be meaningful to improve readability.
  • ANSI standard compilers recognize up to 31 characters. Some may recognize more, but 31 is a safe limit for portability. (Older compilers might only recognize the first 8).
  • Invalid identifiers example: `5cb` (starts with digit), `int` (keyword), `res#` (contains special char).

4. Constants

ConstantsFixed values that do not change during the execution of a program. are values that remain fixed. C supports several types of constants:

  • Primary Constants:
    • Integer Constants: Whole numbers without a decimal point.
      • Must have at least one digit.
      • No decimal point.
      • Can be positive, negative, or zero (default is positive).
      • No commas or blanks allowed within the number (e.g., `1,000` is invalid, use `1000`).
      • Range typically depends on the compiler and system (e.g., -32768 to +32767 for a 2-byte int, or -2147483648 to +2147483647 for a 4-byte int).
      • Examples: `426`, `+782`, `-8000`.
    • Real (Floating-Point) Constants: Numbers with a decimal point or an exponent.
      • Must have at least one digit.
      • Must have a decimal point OR an exponent part (e.g., `3.14`, `2.0e5`, `100.`).
      • Can be positive or negative (default is positive).
      • No commas or blanks.
      • Fractional form: `+325.34`, `426.0`, `-32.76`.
      • Exponential form (mantissa `e` exponent): `3.2e-5` (means 3.2 x 10-5). The mantissa and exponent can be signed.
    • Character Constants: A single character enclosed in single quotes (e.g., `'A'`, `'5'`, `' '`).
  • Secondary Constants (Derived Types): Arrays, Pointers, Structures, Unions, Enums etc. These are more complex and will be covered later.

6. Variables and Data Types

Variables

VariablesA named memory location used to store data that can change during program execution. in C have the same meaning as variables in algebra. A variable is a storage unit that sets aside memory to hold a value, and that value can change during program execution.

Rules to Construct a Valid Variable Name: (Same as for Identifiers)
  • May consist of letters, digits, and underscore (`_`).
  • Must begin with a letter or underscore.
  • ANSI standard recognizes a length of 31 characters (older systems might be less).
  • Case-sensitive (e.g., `total` is different from `TOTAL`).
  • Cannot be a C keyword.
Declaration of Variables: Variables must be declared before they are used. 

// Syntax: data_type variable_name1, variable_name2, ..., variable_nameN;
int age;
float salary, bonus;
char initial;
Assigning Values to Variables: Using the assignment operator (`=`). 

// Syntax: variable_name = constant_or_expression;
int count;
count = 10;     // Assignment
float price = 19.99; // Declaration and initialization
char grade = 'A';
Types of Variables (based on scope and lifetime):
  • Local Variables: Declared inside a function or block. Accessible only within that function/block.
  • Global Variables: Declared outside all functions. Accessible from any part of the program.
  • Static Variables: Retain their value between function calls. Declared with the `static` keyword.

7. Input and Output (I/O)

C provides standard library functions for input and output operations. The most commonly used are `printf()` for output and `scanf()` for input. These functions are declared in the `` (standard input/output) header file, which must be included.

Standard Files

C programs automatically open three standard files (treated as devices):

  • `stdin`: Standard input (usually keyboard)
  • `stdout`: Standard output (usually screen/console)
  • `stderr`: Standard error (usually screen/console)

Formatted I/O Functions

`printf()` - Formatted Output

The `printf()` function is used to display output on the console (standard output). 


// Syntax:
// printf("format string", argument1, argument2, ...);
// printf("string without format specifiers");

#include 

int main() {
    int age = 25;
    float salary = 55000.75;
    char grade = 'A';
    char name[] = "Alex";

    printf("Hello, World!\n"); // Simple string output
    printf("My name is %s and I am %d years old.\n", name, age);
    printf("My grade is %c and salary is %.2f\n", grade, salary);
    // %.2f means print float with 2 decimal places

    return 0;
}
Common format specifiers for `printf()`: `%d` (int), `%c` (char), `%f` (float), `%lf` (double - though `%f` often works), `%s` (string), `%u` (unsigned int), `%ld` (long int), `%p` (pointer address).

`scanf()` - Formatted Input

The `scanf()` function is used to read input from the console (standard input).


// Syntax: scanf("format string", &variable1, &variable2, ...);
// Note the '&' (address-of operator) before variable names (except for strings/char arrays).

#include 

int main() {
    int age;
    float height;
    char initial;
    char fullName[50]; // Character array for string

    printf("Enter your age: ");
    scanf("%d", &age); // Read an integer

    printf("Enter your height (in meters): ");
    scanf("%f", &height); // Read a float

    printf("Enter your first initial: ");
    scanf(" %c", &initial); // Note the space before %c to consume leftover newline

    printf("Enter your full name: ");
    scanf("%s", fullName); // For strings, no & is needed for char array name
                           // scanf with %s reads until whitespace

    printf("\n--- Your Details ---\n");
    printf("Age: %d\n", age);
    printf("Height: %.2f meters\n", height);
    printf("Initial: %c\n", initial);
    printf("Full Name: %s\n", fullName);

    return 0;
}

Important notes for `scanf()`:

  • Always use the address-of operator `&` before variable names for basic types (int, float, char, double) so `scanf` knows *where* to store the input. For character arrays (strings), the array name itself acts as a pointer, so `&` is not used.
  • `scanf("%s", ...)` stops reading at the first whitespace character (space, tab, newline). To read a full line with spaces, use `fgets()`.
  • When mixing `scanf()` for numbers and characters, leftover newline characters in the input buffer can cause issues. A common trick is to put a space before `%c` in `scanf(" %c", &initial);` to consume any leading whitespace, including newlines.

Unformatted I/O Functions (Character I/O)

  • `getchar()`: Reads a single character from `stdin`. `int c = getchar();`
  • `putchar()`: Writes a single character to `stdout`. `putchar(c);`
  • `getc()`: Similar to `getchar()`, but can read from any input stream. `getc(stdin);`
  • `putc()`: Similar to `putchar()`, but can write to any output stream. `putc(c, stdout);`
  • `getch()`: Reads a character directly from the keyboard without echoing it to the screen. Non-standard, often in ``.
  • `getche()`: Reads a character and echoes it. Non-standard, often in ``.

Unformatted I/O Functions (String I/O)

  • `gets()`: Reads a string from `stdin` until a newline is encountered. Warning: `gets()` is unsafe as it doesn't check buffer sizes and can lead to buffer overflows. Avoid using `gets()`. Use `fgets()` instead.
  • `puts()`: Writes a string to `stdout`, followed by a newline character. `puts(myString);`
  • `fgets()`: Reads a string from a stream (like `stdin`) safely. `fgets(str, size, stdin);`

8. Comments and Escape Sequences

Comments in C

CommentsExplanatory notes in the code ignored by the compiler, used to improve readability and document the code. are used to explain the code and make it more readable. The C compiler ignores comments.

  • Single-line comment (from C99 standard): Starts with `//` and extends to the end of the line. 
    // This is a single-line comment
int x = 10; // Variable initialization
  • Multi-line comment (traditional C): Starts with `/*` and ends with `*/`. Can span multiple lines. 
    
/*
This is a
multi-line comment.
It can be used for longer explanations.
*/
int y = 20; /* This is also a multi-line comment on a single line */

Comments cannot be nested within traditional `/* ... */` comments, but `//` can be inside `/* ... */`.

Escape Sequences

Escape sequencesSpecial character combinations starting with a backslash (`\`) used to represent non-printable characters or characters with special meaning within strings and character constants. are used to represent certain special characters within string literals and character constants.

Sequence Meaning
`\n` Newline (moves cursor to the beginning of the next line)
`\t` Horizontal Tab (moves cursor to the next tab stop)
`\r` Carriage Return (moves cursor to the beginning of the current line)
`\b` Backspace (moves cursor one position back)
`\\` Backslash character (`\`)
`\'` Single Quote character (`'`)
`\"` Double Quote character (`"`)
`\?` Question Mark character (`?`)
`\a` Alert or Bell (produces an audible or visible alert)
`\f` Form Feed (moves cursor to the start of the next logical page)
`\v` Vertical Tab
`\0` Null character (marks the end of a string) 

#include 

int main() {
    printf("Hello\tWorld!\n"); // Output: Hello   World! (new line after)
    printf("This is a line.\nAnd this is another line.\n");
    printf("She said, \"Hi!\"\n"); // Output: She said, "Hi!"
    printf("Path: C:\\Users\\YourName\n"); // Output: Path: C:\Users\YourName
    return 0;
}

9. Your First C Program: "Hello, World!"

Let's put some of these concepts together to write a classic "Hello, World!" program.


#include   // Preprocessor directive to include standard input/output functions
#include   // (Often used in older DOS-based compilers like Turbo C for getch())
                     // Not standard C, may not be available on all systems.

// main() function: where program execution begins
void main() {
    // printf() function: displays the string inside quotations
    printf("Hello C Language");

    // getch(): waits for a key press before closing the console window
    // (Primarily for older IDEs/compilers on Windows like Turbo C/C++)
    // For modern compilers/systems, this is often not needed or getchar() can be used.
    getch();
}

/*
Explanation:
1. #include : Includes the standard input/output library, which gives us access to functions like printf().
2. #include : (Console Input/Output) Is often included in older learning environments for functions like getch().
   Modern C development usually doesn't require conio.h. For pausing, getchar() from stdio.h can be used.
3. void main(): This is the main function. `void` means it doesn't return any value.
   Standard C prefers `int main()` which returns an integer.
4. printf("Hello C Language");: This line prints the text "Hello C Language" to the console.
5. getch();: This function (from conio.h) waits for the user to press any key. It's often used in older console
   environments to prevent the output window from closing immediately after the program finishes.
   If conio.h is not available, you can remove this line or use getchar(); (which requires pressing Enter).
*/
Standard Compliant "Hello, World!": 

#include 

int main(void) { // `int main()` is standard, `void` indicates no command-line arguments
    printf("Hello, World!\n"); // `\n` adds a newline for better formatting
    return 0; // Indicates successful execution to the operating system
}
This version is more portable and follows modern C standards.
