Introduction to C

C is a powerful, general-purpose programming language developed in 1972 by Dennis RitchieAn American computer scientist who co-created the C programming language and the Unix operating system. Recipient of the Turing Award. at Bell Laboratories of AT&T. It was initially designed for system programming tasks, particularly for the UNIX operating system.

Known for its efficiency, low-level memory access, and minimal runtime overhead, C provides a foundation for many other programming languages and remains influential in areas like operating systems, embedded systems, game development, and high-performance computing. It strikes a balance between high-level language features (structure, readability) and low-level capabilities (direct hardware interaction).

History & Importance of C

The genesis of C is deeply intertwined with the development of the UNIX operating system:

  • Early Influences (1960s): Rooted in ALGOL 60, CPL (Combined Programming Language), and BCPL (Basic CPL by Martin Richards). Ken Thompson developed 'B', a typeless language based on BCPL, for an early UNIX on the PDP-7.
  • Birth of C (1972): Dennis Ritchie, at Bell Labs, evolved 'B' into 'C' by adding data types (like int, char) and other crucial features.
  • UNIX Rewritten in C (1973): This was a pivotal moment. Rewriting the UNIX kernel in C demonstrated its power for system-level programming and made UNIX highly portable.
  • K&R C (1978): Brian Kernighan and Dennis Ritchie published "The C Programming Language." This book served as the de facto standard for years, often referred to as "K&R C."
  • Standardization (ANSI C & ISO C):
    • C89/C90: The American National Standards Institute (ANSI) standardized C in 1989 (X3.159-1989). This was adopted by ISO in 1990 (ISO/IEC 9899:1990).
    • C99: Introduced new features like inline functions, new data types (e.g., long long int, _Bool), variable-length arrays, and single-line comments (//).
    • C11 & C18/C23: Further refinements including multi-threading support, atomic operations, generic selections, and improved Unicode support. C23 is the latest standard.

Importance of C:

  • Systems Programming: The core of most operating systems (Windows, Linux, macOS, iOS, Android) is written in C.
  • Embedded Systems: Its efficiency and direct hardware control make it ideal for microcontrollers and IoT devices.
  • Performance-Critical Applications: Compilers, interpreters, databases, and game engines often leverage C for speed.
  • Foundation for Other Languages: C's syntax and concepts heavily influenced C++, C#, Java, Objective-C, Perl, Python, and PHP, among others. Learning C provides a strong understanding of fundamental programming principles.

Structure of a C Program

A C program typically adheres to a standard structure, promoting organization and readability:

  1. Preprocessor Directives: Instructions processed before compilation (e.g., #include for header files, #define for macros).
  2. Global Declarations: Variables, functions, and type definitions accessible throughout the program.
  3. The main() function: The mandatory entry point where program execution begins. It can return an integer status to the operating system.
  4. User-Defined Functions: Custom functions to perform specific tasks, enhancing modularity.
  5. Comments: Explanations for human readers, ignored by the compiler.

A canonical "Hello, World!" example illustrates this structure:

#include    // 1. Preprocessor Directive: Includes Standard Input/Output library

// 2. Global declarations (none in this simple example)

// 3. The main() function: Program execution starts here
int main() {
    // Local variable declaration (none in this simple example)
    
    // Statement: Calls printf function to display output
    printf("Hello, World!\n"); 

    return 0;  // Indicates successful execution
}

// 4. User-defined functions (none in this simple example)

Compilation & Execution Process

Transforming C source code into an executable program involves a multi-stage process, typically managed by a C compiler toolchain (like GCC or Clang):

  1. Preprocessing (cpp): The preprocessor handles directives. It expands macros (#define), includes header files (#include), and processes conditional compilation (#if, #ifdef). The output is an expanded C source file (often with a .i extension).
  2. Compilation (cc or ccl): The compiler translates the preprocessed source code into assembly language specific to the target architecture. This assembly code is human-readable but low-level. (Output: .s or .asm file).
  3. Assembly (as): The assembler converts the assembly code into machine code (binary instructions). This results in an object file (.o or .obj), which contains relocatable machine code and information for the linker.
  4. Linking (ld): The linker takes one or more object files (from your code and pre-compiled libraries) and resolves symbol references (like function calls between files or to library functions). It combines these into a single executable file (e.g., a.out on Unix-like systems, .exe on Windows).

Once the executable is created, it can be loaded into memory by the operating system and executed by the CPU.

Keywords & Identifiers

Keywords

Keywords are predefined, reserved words in C that have special meanings to the compiler. They cannot be used as names for variables, functions, or other user-defined identifiers.

Standard C (C89/C90) has 32 keywords. Subsequent standards (C99, C11, C23) have added a few more (e.g., inline, _Bool, _Generic, nullptr).

Fig 4: The original 32 keywords of the C language (ANSI C89/ISO C90).

Identifiers

Identifiers are user-defined names for program elements like variables, constants (const variables), functions, arrays, structures, etc. Rules for naming identifiers:

  • Must consist of letters (a-z, A-Z), digits (0-9), and underscores (_).
  • The first character must be a letter or an underscore. (Identifiers starting with an underscore, especially two underscores or an underscore followed by an uppercase letter, are often reserved for compiler/library implementation ‚Äì best to avoid for general use).
  • Cannot be a C keyword.
  • C is case-sensitive: myVariable and myvariable are distinct identifiers.
  • Should be descriptive and meaningful. While ANSI C guarantees significance for at least the first 31 characters for external identifiers (and often more for internal), modern compilers typically support much longer names.

Valid examples: count, _tempValue, studentName, MAX_USERS, calculateTotal_1.

Invalid examples: 1stPlace (starts with a digit), net-salary (contains hyphen), int (keyword), my Variable (contains space).

Constants

Constants are fixed values that the program cannot alter during its execution. They are also known as literals.

  • Integer Constants: Whole numbers like 10, -200, 0.
    • Decimal: 123
    • Octal (prefix 0): 0173 (decimal 123)
    • Hexadecimal (prefix 0x or 0X): 0x7B (decimal 123)
    • Suffixes: U (unsigned), L (long), LL (long long) - e.g., 100U, 12345L, 987ULL.
  • Floating-Point Constants: Real numbers with a decimal point or an exponent, like 3.14, -0.005, 2.5e4 (2.5 x 104).
    • Suffixes: f or F (float), l or L (long double) - e.g., 3.14f, 1.23456789L. Default is double.
  • Character Constants: A single character enclosed in single quotes, e.g., 'A', '7', '$', '\n' (escape sequence). Stored as their integer (ASCII/Unicode) value.
  • String Constants (String Literals): A sequence of characters enclosed in double quotes, e.g., "Hello, C!". Automatically appended with a null character (\0) by the compiler.

Constants can also be defined symbolically using:

  1. #define Preprocessor Directive: Creates macro constants (textual substitution). 
    #define PI 3.14159
  2. const Keyword: Creates typed constants (variables whose values cannot be changed after initialization). This is generally preferred over #define for type safety. 
    const float EULER_NUMBER = 2.71828f;
const int MAX_ATTEMPTS = 3;

Variables

A variable is a named memory location used to store data that can be modified during program execution. Each variable in C has a specific data type, which determines the size and layout of the variable's memory, the range of values that can be stored, and the set of operations that can be applied to it.

Declaration:

A variable must be declared before it can be used. The declaration specifies the variable's type and name.

Syntax: data_type variable_name1, variable_name2, ...;

int userAge;          // Declares an integer variable named userAge
float accountBalance;   // Declares a float variable
char middleInitial;     // Declares a character variable
double piApproximation; // Declares a double-precision float

Initialization:

Assigning an initial value to a variable at the time of its declaration.

int score = 0;                    // Declares and initializes score to 0
float itemPrice = 19.99f;         // Initializes itemPrice
char userChoice = 'Y';             // Initializes userChoice
const int DAYS_IN_WEEK = 7;   // A constant variable, must be initialized

Assignment:

Storing or changing the value of an already declared variable using the assignment operator (=).

userAge = 25;
accountBalance = 1234.50;
middleInitial = 'D';

Data Types

Data types define the nature of the data a variable can hold and the operations that can be performed on it. C offers a rich set of built-in (or fundamental) data types, which can be broadly categorized:

Primary Data Types:

Type Description Typical Size (bytes) Typical Range / Precision printf Specifier
char Single character or small integer 1 signed: -128 to 127, unsigned: 0 to 255 %c (char), %d (as int)
int Integer values System dependent (often 4) e.g., for 4 bytes: -2,147,483,648 to 2,147,483,647 %d, %i
float Single-precision floating-point 4 Approx. 6-7 decimal digits precision %f, %e, %g
double Double-precision floating-point 8 Approx. 15-16 decimal digits precision %lf, %le, %lg (for scanf always %lf)
void Represents "no type". Used for functions that return nothing or for generic pointers. N/A N/A N/A

Type Modifiers:

These alter the meaning or range of the basic types:

  • signed: Explicitly indicates the type can hold positive and negative values (default for int, char can vary).
  • unsigned: Restricts the type to hold only non-negative values, effectively doubling the positive range. (e.g., unsigned int).
  • short: May use less storage than int (e.g., short int, often 2 bytes). Format: %hd.
  • long: May use more storage than int (e.g., long int, often 4 or 8 bytes). Format: %ld.
  • long long (C99+): Even larger integer type (e.g., long long int, typically 8 bytes). Format: %lld.

Example: unsigned long int counter;

Derived Data Types:

Constructed from fundamental types: Arrays, Pointers, Structures, Unions.

User-Defined Data Types:

enum (Enumerations), typedef (Type Definitions).

Input & Output (using scanf, printf)

The C standard library provides functions for input and output operations, primarily through .

printf() - Formatted Output

The printf() function sends formatted output to the standard output stream (usually the console).

Syntax: int printf(const char *format_string, ...);

The format_string contains literal text and format specifiersPlaceholders like %d, %f, %c that are replaced by the values of subsequent arguments.. It returns the number of characters printed.

#include 

int main() {
    int age = 30;
    float pi = 3.14159f;
    char grade = 'A';
    char name[] = "Alice";

    printf("Hello, %s!\n", name);
    printf("Age: %d, Grade: %c\n", age, grade);
    printf("Value of PI: %.2f\n", pi); // Prints PI with 2 decimal places
    
    return 0;
}

scanf() - Formatted Input

The scanf() function reads formatted input from the standard input stream (usually the keyboard).

Syntax: int scanf(const char *format_string, ...);

It reads data according to the format_string and stores it in the memory locations pointed to by the subsequent arguments (which must be pointers, hence the & operator for variables). It returns the number of items successfully read.

Crucial Note: Always use the address-of operator (&) with variables in scanf (except for strings/char arrays, where the array name itself decays to a pointer).

Output/Simulation will appear here. (Note: True scanf interaction is complex in a browser. This simulates a typical interaction.)

Common Pitfall with scanf: When reading a character (%c) after a number (%d or %f), the newline character left in the input buffer by the previous scanf might be consumed by %c. To avoid this, put a space before %c in the format string: scanf(" %c", &myChar);. This tells scanf to skip any leading whitespace, including newlines.

Comments in C

Comments are non-executable text used to explain code, making it more understandable for humans. The C compiler ignores them.

  • Single-line comments (C99+): Start with // and extend to the end of the line. 
    int x = 10; // This is a single-line comment explaining variable x
  • Multi-line comments (Traditional C): Start with /* and end with */. They can span multiple lines. 
    /*
 This is a multi-line comment.
 It can provide more detailed explanations
 about a block of code or a complex function.
*/
float calculate_area(float radius);

Nesting Comments: Standard C does not allow nesting of multi-line comments (/* ... /* ... */ ... */ is invalid). However, single-line comments can be inside multi-line comments, and multi-line comment delimiters can "comment out" single-line comments.

Good commenting practices improve code maintainability and collaboration.

Escape Sequences

Escape sequences are special character combinations, beginning with a backslash (\), used within character and string literals to represent non-printable characters or characters that have special meaning in C.

Sequence Meaning Description
\n Newline Moves the cursor to the beginning of the next line.
\t Horizontal Tab Moves the cursor to the next horizontal tab stop.
\r Carriage Return Moves the cursor to the beginning of the current line (without advancing to the next line).
\b Backspace Moves the cursor one position back.
\\ Backslash Represents a literal backslash character.
\' Single Quote Represents a literal single quote character (used within character constants).
\" Double Quote Represents a literal double quote character (used within string literals).
\0 Null Character Represents the null terminator, often used to mark the end of strings. Has an integer value of 0.
\a Alert (Bell) Produces an audible or visible alert (system dependent).
\? Question Mark Represents a literal question mark (useful for avoiding trigraphs, though trigraphs are largely obsolete).
\v Vertical Tab Moves the cursor to the next vertical tab stop.
\f Form Feed Moves the cursor to the initial position of the next logical page.
\xhh Hexadecimal value Represents a character by its hexadecimal ASCII value (e.g., \x41 is 'A').
\ooo Octal value Represents a character by its octal ASCII value (e.g., \101 is 'A').
