Operators and Expressions in C

Understanding how C manipulates data and performs calculations.

1. Introduction to Operators and Expressions

An OperatorA symbol that tells the compiler to perform specific mathematical or logical manipulations. is a symbol that performs an operation on one or more operandsThe data items (constants or variables) on which an operator acts.. For example, in `a + b`, `a` and `b` are operands, and `+` is the operator.
An ExpressionA sequence of operands and operators that reduces to a single value. is a combination of operators and operands that, when evaluated, produces a single value.
  • A simple expression contains only one operator (e.g., `3 + 5`, `-a`).
  • A complex expression contains more than one operator (e.g., `6 + 8 * 7`).

Arithmetic operators perform mathematical operations such as addition, subtraction, multiplication, division, and modulus (remainder) on numerical values (constants and variables).

/ Division a / b 2 (integer division)

#include 

int main() { // Standard C uses int main()
    int a = 9, b = 4, c;

    c = a + b;
    printf("a + b = %d\n", c); // Output: a + b = 13

    c = a - b;
    printf("a - b = %d\n", c); // Output: a - b = 5

    c = a * b;
    printf("a * b = %d\n", c); // Output: a * b = 36

    c = a / b; // Integer division, fractional part is truncated
    printf("a / b = %d\n", c); // Output: a / b = 2

    c = a % b;
    printf("Remainder when a divided by b = %d\n", c); // Output: Remainder when a divided by b = 1

    // For floating point division
    float x = 9.0, y = 4.0, z;
    z = x / y;
    printf("x / y (float) = %f\n", z); // Output: x / y (float) = 2.250000

    return 0;
}

Note on Division (`/`): If both operands are integers, integer division is performed (the fractional part is discarded). To get a floating-point result, at least one operand must be a float or double, or type casting must be used.

Relational operators check the relationship between two operands. If the relation is true, it returns `1`; if the relation is false, it returns `0`. They are commonly used in decision-makinge.g., in `if` statements and loopse.g., in `while` or `for` loop conditions.

Operator Meaning Example Return Value
<< /td> is less than 2 < 9 1 (true)
<=< /td> is less than or equal to 2 <= 2 1 (true)
> is greater than 2 > 9 0 (false)
>= is greater than or equal to 3 >= 2 1 (true)
== is equal to 2 == 3 0 (false)
!= is not equal to 2 != 2 0 (false) 

#include 

int main() {
    int a = 5, b = 5, c_val = 10; // Renamed c to c_val to avoid conflict

    printf("%d == %d is %d\n", a, b, a == b);     // 5 == 5 is 1 (true)
    printf("%d == %d is %d\n", a, c_val, a == c_val); // 5 == 10 is 0 (false)

    printf("%d > %d is %d\n", a, b, a > b);      // 5 > 5 is 0 (false)
    printf("%d > %d is %d\n", a, c_val, a > c_val);   // 5 > 10 is 0 (false)

    printf("%d < %d is %d\n", a, b, a < b);      // 5 < 5 is 0 (false)
    printf("%d < %d is %d\n", a, c_val, a < c_val);   // 5 < 10 is 1 (true)

    printf("%d != %d is %d\n", a, b, a != b);     // 5 != 5 is 0 (false)
    printf("%d != %d is %d\n", a, c_val, a != c_val); // 5 != 10 is 1 (true)

    printf("%d >= %d is %d\n", a, b, a >= b);     // 5 >= 5 is 1 (true)
    printf("%d >= %d is %d\n", a, c_val, a >= c_val); // 5 >= 10 is 0 (false)

    printf("%d <= %d is %d\n", a, b, a <= b);     // 5 <= 5 is 1 (true)
    printf("%d <= %d is %d\n", a, c_val, a <= c_val); // 5 <= 10 is 1 (true)

    return 0;
}

Logical operators are used to combine or invert the results of two or more conditions (expressions evaluating to true/1 or false/0). An expression containing logical operators returns either `0` (false) or `1` (true).

Operator Meaning Example Return Value
&& Logical AND (9 > 2) && (17 > 2) 1 (true)
|| Logical OR (9 > 2) || (17 == 7) 1 (true)
! Logical NOT !(29 != 29) 1 (true, because 29!=29 is false)

Truth Tables:

Logical AND (&&): Returns true (1) only if all conditions are true.

true (1) true (1) true (1)
true (1) false (0) false (0)
false (0) true (1) false (0)
false (0) false (0) false (0)
Logical OR (||): Returns true (1) if at least one condition is true.
Operand1 Operand2 Operand1 || Operand2
true (1) true (1) true (1)
true (1) false (0) true (1)
false (0) true (1) true (1)
false (0) false (0) false (0)
Logical NOT (!): Reverses the logical state of its operand.
Operand1 !Operand1
true (1) false (0)

#include 

int main() {
    int a = 5, b = 5, c_val = 10;
    int result;

    // Logical AND
    result = (a == b) && (c_val > b); // (true) && (true) -> true
    printf("(a == b) && (c_val > b) is %d\n", result); // Output: 1

    result = (a == b) && (c_val < b); // (true) && (false) -> false
    printf("(a == b) && (c_val < b) is %d\n", result); // Output: 0

    // Logical OR
    result = (a == b) || (c_val < b); // (true) || (false) -> true
    printf("(a == b) || (c_val < b) is %d\n", result); // Output: 1

    result = (a != b) || (c_val < b); // (false) || (false) -> false
    printf("(a != b) || (c_val < b) is %d\n", result); // Output: 0

    // Logical NOT
    result = !(a == b); // !(true) -> false
    printf("!(a == b) is %d\n", result); // Output: 0

    result = !(a != b); // !(false) -> true
    printf("!(a != b) is %d\n", result); // Output: 1

    return 0;
}

Short-circuit evaluationFor `&&`, if the first operand is false, the second is not evaluated. For `||`, if the first is true, the second is not evaluated. is an important feature of logical AND and OR operators.

Assignment operators are used to assign a value (or the result of an expression) to a variable.

Simple Assignment Operator (`=`): Assigns the value from the right-hand side to the left-hand side variable.
Syntax: `variable_name = expression_or_value;`
Examples: `x = 10;`, `y = a + b;`, `z = p;`

Compound Assignment Operators:

int main() {
    int a = 5, c_var;

    c_var = a;      // Simple assignment
    printf("c_var (after c_var = a) = %d\n", c_var);         // Output: 5

    printf("c_var (after c_var -= a) = %d\n", c_var);        // Output: 5

    c_var *= a;     // c_var = c_var * a  => c_var = 5 * 5
    printf("c_var (after c_var *= a) = %d\n", c_var);        // Output: 25

    c_var /= a;     // c_var = c_var / a  => c_var = 25 / 5
    printf("c_var (after c_var /= a) = %d\n", c_var);        // Output: 5

}

These are unary operatorsThey operate on a single operand. that add or subtract 1 from their operand, respectively. They can only be applied to variables, not constants or expressions. They have high precedence.

Operator Name Description
++x Pre-increment Increments x, then uses the new value of x in the expression.
x++ Post-increment Uses the current value of x in the expression, then increments x.
--x Pre-decrement Decrements x, then uses the new value of x in the expression.
x-- Post-decrement Uses the current value of x in the expression, then decrements x. 

#include 

int main() {
    int a = 10, b = 100;
    float c_val = 10.5, d_val = 10.5;
    int pre_inc, post_inc, pre_dec, post_dec;

    // Pre-increment
    pre_inc = ++a; // a becomes 11, pre_inc becomes 11
    printf("Pre-increment: ++a = %d, a = %d\n", pre_inc, a);

    // Post-increment
    // Reset a for clarity
    a = 10;
    post_inc = a++; // post_inc becomes 10, a becomes 11
    printf("Post-increment: a++ = %d, a = %d\n", post_inc, a);

    // Pre-decrement
    pre_dec = --b; // b becomes 99, pre_dec becomes 99
    printf("Pre-decrement: --b = %d, b = %d\n", pre_dec, b);

    // Post-decrement
    // Reset b for clarity
    b = 100;
    post_dec = b--; // post_dec becomes 100, b becomes 99
    printf("Post-decrement: b-- = %d, b = %d\n", post_dec, b);


    // With floats
    printf("Original c_val = %f\n", c_val);
    printf("++c_val = %f\n", ++c_val); // c_val becomes 11.5

    printf("Original d_val = %f\n", d_val);
    printf("d_val++ = %f\n", d_val++); // uses 10.5, then d_val becomes 11.5
    printf("d_val after post-increment = %f\n", d_val);

    return 0;
}

Behavior in Complex Expressions (e.g., `printf` arguments):

The order of evaluation of arguments to a function like `printf` is unspecified by the C standardCompilers can evaluate arguments in any order, typically right-to-left for many common compilers on x86 architecture. Relying on a specific order is non-portable.. This can lead to confusing results when multiple increment/decrement operators are used on the same variable within such arguments.

Example from your data: `printf("%d %d %d", i, ++i, i++);` with `i=1` might give `3 3 1` on some compilers.

  • If arguments are evaluated right-to-left:
    1. `i++`: `i` (1) is used for `printf`, then `i` becomes 2.
    2. `++i`: `i` (2) becomes 3, then `i` (3) is used for `printf`.
    3. `i`: `i` (3) is used for `printf`.
    So, `printf` receives `3, 3, 1` to print.

Similarly, `j = i++ + ++i;` involves undefined behaviorModifying a variable more than once between two sequence points without a defined order. before C++17 (and still tricky in C). It's best to avoid such complex expressions where the order of side effects is critical and unclear.

The conditional operator is a ternary operatorIt takes three operands. and is a shorthand way of writing an `if-else` statement.
Syntax: `condition ? expression_if_true : expression_if_false;`

It first evaluates the `condition`.

  • If `condition` is true (non-zero), `expression_if_true` is evaluated, and its value becomes the result of the overall conditional expression.
  • If `condition` is false (zero), `expression_if_false` is evaluated, and its value becomes the result.


#include 

int main() {
    int a = 10, b = 20;
    int max;

    max = (a > b) ? a : b; // If a > b, max is a, else max is b
    printf("The maximum of %d and %d is %d\n", a, b, max); // Output: 20

    char february_status;
    int days;
    printf("Is this a leap year? (Enter 'L' for Leap, 'N' for Not Leap): ");
    scanf(" %c", &february_status); // Space to consume potential newline

    // Using char 'L' or 'N'
    // days = (february_status == 'L' || february_status == 'l') ? 29 : 28;

    // Example based on your image (using 1 for leap)
    int is_leap_input;
    printf("If this year is leap year, enter 1. If not enter any other integer: ");
    scanf("%d", &is_leap_input);
    days = (is_leap_input == 1) ? 29 : 28;
    printf("Number of days in February = %d\n", days);


    return 0;
}

Bitwise Complement (~)

This is a unary operator. It flips all the bits of its operand (0 becomes 1, and 1 becomes 0). For any integer `n`, the bitwise complement `~n` will be `-(n+1)` due to how negative numbers are represented using Two's ComplementA method for representing signed integers where negative numbers are the bitwise complement of the positive number, plus one..

Two's complement of a number is (One's complement) + 1.
So, `~n` (One's complement) is equivalent to `(-n) - 1` in value for positive `n` when considering signed representation. More generally, `~n = -(n+1)`.

Example: `~35` 
35 = 00100011 (assuming 8-bit for simplicity)
~35 = 11011100 (One's complement)
This value, if interpreted as a signed 8-bit number using two's complement, represents -36.
Because: -(35+1) = -36.
                

#include 
int main() {
    int num1 = 35;
    int num2 = -12;
    printf("Complement of %d = %d\n", num1, ~num1); // Output: Complement of 35 = -36
    printf("Complement of %d = %d\n", num2, ~num2); // Output: Complement of -12 = 11
    return 0;
}

Shift Left (<<)< /h4>

Shifts all bits of the first operand to the left by the number of positions specified by the second operand. Vacated bits on the right are filled with zeros.

`x << n` is equivalent to multiplying `x` by 2n. 

#include 
int main() {
    unsigned char a = 5; // 00000101
    printf("%d << 1 = %d\n", a, a << 1); // 00001010 (10)
    printf("%d << 2 = %d\n", a, a << 2); // 00010100 (20)
    return 0;
}

Shift Right (>>)

Shifts all bits of the first operand to the right by the number of positions specified by the second operand.

  • For unsigned numbersAlways positive., vacated bits on the left are filled with zeros (logical right shift).
  • For signed numbersCan be positive or negative., the behavior can be implementation-defined:
    • Arithmetic Right Shift: Vacated bits on the left are filled with the sign bit (to preserve the sign). This is common.
    • Logical Right Shift: Vacated bits on the left are filled with zeros.
`x >> n` is roughly equivalent to dividing `x` by 2n (for positive `x`).


#include 
int main() {
    unsigned char a = 20; // 00010100
    signed char b = -20;  // e.g., 11101100 in 8-bit 2's complement
                          // (Exact bit pattern depends on system int size)

    printf("%u >> 1 = %u\n", a, a >> 1); // 00001010 (10)
    printf("%u >> 2 = %u\n", a, a >> 2); // 00000101 (5)

    // For signed, assuming arithmetic shift (common)
    // -20 >> 1 should be -10
    printf("%d >> 1 = %d\n", b, b >> 1);
    return 0;
}

Comma Operator (,)

The comma operator is used to link related expressions together. It evaluates expressions from left to right, and the value of the rightmost expression becomes the value of the combined comma-separated expression.

Example: `val = (a=3, b=9, c=77, a+c);`
  1. `a` is assigned `3`.
  2. `b` is assigned `9`.
  3. `c` is assigned `77`.
  4. `a+c` (i.e., `3+77 = 80`) is evaluated.
  5. The value `80` is assigned to `val`.

It's often used in `for` loops to include multiple initializations or updates in a single part of the loop syntax.


#include 
int main() {
    int x, y;
    x = (y = 5, y + 10); // y becomes 5, then y+10 (15) is assigned to x
    printf("x = %d, y = %d\n", x, y); // Output: x = 15, y = 5
    return 0;
}

`sizeof` Operator

The `sizeof` operator is a unary operator that returns the size, in bytes, of its operand (which can be a variable, a data type, or an expression). It's very useful for determining memory allocation and ensuring portability.

Syntax: `sizeof(operand)` or `sizeof operand` (if operand is not a type name). 

#include 

int main() {
    int a;
    int e[10]; // Array of 10 integers
    float b_f; // Renamed to avoid conflict with int b in other examples
    double c_d;
    char d_ch;

    // Use %zu for sizeof result as it returns size_t type
    printf("Size of int = %zu bytes\n", sizeof(a));
    printf("Size of int type = %zu bytes\n", sizeof(int));
    printf("Size of float = %zu bytes\n", sizeof(b_f));
    printf("Size of double = %zu bytes\n", sizeof(c_d));
    printf("Size of char = %zu bytes\n", sizeof(d_ch));
    printf("Size of integer type array having 10 elements = %zu bytes\n", sizeof(e));
    // Output will vary based on system architecture, e.g.:
    // Size of int = 4 bytes
    // Size of int type = 4 bytes
    // Size of float = 4 bytes
    // Size of double = 8 bytes
    // Size of char = 1 byte
    // Size of integer type array having 10 elements = 40 bytes (if int is 4 bytes)

    return 0;
}

Note: The correct format specifier for the `size_t` type returned by `sizeof` is `%zu` (C99 and later). If using an older compiler, `%lu` (unsigned long) might be used, but `%zu` is preferred for portability.

11. Operator Precedence and Associativity

When an expression contains multiple operators, C uses rules of precedenceDetermines which operator is performed first in an expression. Higher precedence operators are evaluated before lower precedence ones. and associativityDetermines the order in which operators of the same precedence are evaluated (e.g., left-to-right or right-to-left). to determine the order of evaluation.

Rules for Evaluation of Expression:

  1. Sub-expressions in parentheses `()` are evaluated first. Nested parentheses are evaluated from innermost to outermost.
  2. Operators are then evaluated according to their precedence.
  3. If operators have the same precedence, their associativity determines the order.
    • Left-to-Right (L-R): Operators are evaluated from left to right. Most binary operators in C have L-R associativity.
    • Right-to-Left (R-L): Operators are evaluated from right to left. Unary operators, assignment operators, and the conditional operator typically have R-L associativity.

Operator Precedence and Associativity Table (Simplified Common Operators):

Precedence Operator Description Associativity
1 (Highest) () [] . -> ++ -- (postfix) Parentheses, Array subscript, Member selection L-R
2 ++ -- (prefix) + - (unary) ! ~ * & (address) sizeof (type) Unary, Logical NOT, Bitwise NOT, Dereference, Address-of, Sizeof, Type cast R-L
3 * / % Multiplication, Division, Modulus L-R
4 + - (binary) Addition, Subtraction L-R
5 <<>> Bitwise Left Shift, Right Shift L-R
6 < <=> >= Relational L-R
7 == != Equality L-R
8 & (bitwise) Bitwise AND L-R
9 ^ (bitwise) Bitwise XOR L-R
10 | (bitwise) Bitwise OR L-R
11 && Logical AND L-R
12 || Logical OR L-R
13 ? : Conditional (Ternary) R-L
14 = += -= *= /= %= &= ^= |= <<=>>= Assignment Operators R-L
15 (Lowest) , Comma Operator L-R

Example: `a = x - y / 3 + z * 2 - p / 4;` with `x=7, y=9, z=11, p=8`.
Evaluation according to precedence:

  1. `y / 3` (9/3 = 3)
  2. `z * 2` (11*2 = 22)
  3. `p / 4` (8/4 = 2)
  4. Expression becomes: `a = 7 - 3 + 22 - 2`
  5. `7 - 3` (4) (L-R associativity for `-` and `+`)
  6. `4 + 22` (26)
  7. `26 - 2` (24)
  8. `a = 24`
If parentheses were used: `(9 - 12 / (3 + 3)) * (2 - 1)`
  1. Innermost `(3+3)` = 6
  2. `12 / 6` = 2
  3. `9 - 2` = 7 (Left parenthesis done)
  4. `(2-1)` = 1 (Right parenthesis done)
  5. `7 * 1` = 7

12. Type Conversions (Type Casting)

Type conversion is the process of converting a value from one data type to another. C allows both implicit and explicit type conversions.

Implicit Type Conversion (Automatic Conversion / Type Promotion)

Performed automatically by the compiler when operands of different types are used in an expression, or when a value of one type is assigned to a variable of another type, if the conversion is "safe" (e.g., from a smaller type to a larger type without loss of precision).

Integer Promotion: Values of integral types "smaller" than `int` or `unsigned int` (like `char`, `short int`) are promoted to `int` or `unsigned int` before any operation is performed on them. 


#include 
int main() {
    char a = 'A'; // ASCII value of 'A' is 65
    int b = 10;
    int sum = a + b; // 'a' is promoted to int (65)
    printf("Sum = %d\n", sum); // Output: Sum = 75
    return 0;
}

Usual Arithmetic Conversions: When an operation involves operands of different arithmetic types, the "lower" type is promoted to the "higher" type according to a hierarchy to ensure no loss of data. The hierarchy (simplified) is generally: `int` → `unsigned int` → `long int` → `unsigned long int` → `long long int` → `unsigned long long int` → `float` → `double` → `long double`.

Explicit Type Conversion (Type Casting)

Programmer-forced conversion using the cast operator`(type_name)`.
Syntax: `(target_type_name) expression;` 


#include 

int main() {
    int num1 = 9, num2 = 4;
    float result;

    // Without type casting (integer division)
    result = num1 / num2;
    printf("Without casting: %d / %d = %f (incorrect float representation of int 2)\n", num1, num2, result); // Output: 2.000000

    // With type casting
    result = (float)num1 / num2; // num1 is temporarily treated as float
    printf("With casting: (float)%d / %d = %f\n", num1, num2, result); // Output: 2.250000

    // Casting an integer to char
    int ascii_val = 65;
    char character = (char)ascii_val;
    printf("Character for ASCII %d is %c\n", ascii_val, character); // Output: A

    return 0;
}

Common rules/considerations for explicit casting:

  • All integer types can be converted to float.
  • All float types can be converted to double.
  • All character types can be converted to integer.
  • Converting from a larger type to a smaller type (e.g., `double` to `int`) can lead to loss of data (truncation of fractional part, or overflow if the value is too large).
