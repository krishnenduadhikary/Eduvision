Topic 1

Introduction to Image Processing

Image processing is a method to perform operations on an image to enhance it or extract useful information from it. It is a type of signal processing where the input is an image and the output may be either an image or a set of characteristics related to the image.

What is a Digital Image? A digital image is a representation of a two-dimensional image using a finite set of digital values, called picture elements or pixels.

Pixels Color Channels Resolution Bit Depth

Digital image processing allows the use of much more complex algorithms for image processing and hence can offer both more sophisticated performance at simple tasks, and the implementation of methods which would be impossible by analog means.

1.1 Fundamentals of Images

Definition of Digital Image and Image Processing

A digital image is a numeric representation of a two-dimensional image. It is composed of picture elements called pixels, each with a particular location and value. Digital image processing refers to processing digital images by means of a digital computer.

A digital image can be represented as a 2D function f(x,y) where x and y are spatial coordinates, and the amplitude of f at any pair of coordinates (x,y) is called the intensity or gray level of the image at that point.

History and Applications of Image Processing

Historical Timeline
  • 1920s: Early image transmission systems (Bartlane cable picture transmission)
  • 1960s: NASA's Jet Propulsion Laboratory processes space images
  • 1970s: Medical imaging advances (CT scans, MRI)
  • 1980s-Now: Digital revolution, computer vision, AI-based image analysis
Key Applications
  • Medical: MRI, CT scans, X-ray analysis
  • Remote Sensing: Satellite imagery, land use mapping
  • Computer Vision: Face recognition, object detection
  • Industrial: Quality control, defect detection
  • Multimedia: Image enhancement, compression

Image Formation and Acquisition

Images are formed through the interaction of physical energy (light, X-rays, etc.) with sensors. The process involves:

  1. Illumination: Energy source illuminates the scene
  2. Reflection/Transmission: Energy interacts with objects
  3. Sensing: Sensors capture the energy
  4. Digitization: Analog signals are converted to digital form

Common Acquisition Devices: Digital cameras, scanners, medical imaging devices (MRI, CT), satellite sensors, microscopy systems.

Types of Images

Grayscale

Single channel image with intensity values from black (0) to white (255 for 8-bit).

Color (RGB)

Three channels (Red, Green, Blue) combined to create various colors.

Binary

Only two intensity values: 0 (black) and 1 (white). Used for masks and segmentation.

Multispectral

Multiple image bands at specific wavelengths (typically 3-10 bands).

Hyperspectral

Hundreds of contiguous spectral bands for detailed spectral analysis.

Digital Image Properties

Resolution

Number of pixels per unit area. Higher resolution means more detail. Common measures: PPI (pixels per inch), DPI (dots per inch).

Dynamic Range

Range of intensity values a pixel can take. For 8-bit images: 0-255 (256 levels). Higher bit depth = greater dynamic range.

Aspect Ratio

Ratio of image width to height. Common ratios: 4:3 (standard), 16:9 (widescreen), 1:1 (square).

1.2 Image Representation

Continuous vs. Discrete Images

Continuous Images
  • Defined for all spatial coordinates
  • Infinite resolution
  • Analog representation
  • Examples: photographs, paintings
Discrete Images
  • Defined only at discrete points
  • Finite resolution (pixels)
  • Digital representation
  • Examples: digital photos, computer graphics

Sampling and Quantization

Sampling

The process of digitizing the coordinate values (spatial domain).

Sampling Rate: Number of samples taken per unit distance. Higher sampling rate = better spatial resolution.

Continuous Image → Sampled at discrete points → Pixel grid

Quantization

The process of digitizing the amplitude values (intensity domain).

Quantization Levels: Number of discrete intensity values. More levels = smoother intensity transitions.

Continuous intensity → Quantized to discrete levels → Digital values

Nyquist-Shannon Sampling Theorem: To accurately reconstruct a signal, it must be sampled at least twice the highest frequency present in the signal.

Image File Formats

JPEG

Joint Photographic Experts Group

Lossy compression, good for photographs, supports 24-bit color.

PNG

Portable Network Graphics

Lossless compression, supports transparency, good for graphics.

BMP

Bitmap

Uncompressed, simple format, large file size, Windows standard.

TIFF

Tagged Image File Format

Flexible, supports lossless compression, used in publishing.

GIF

Graphics Interchange Format

Lossless, limited to 256 colors, supports animation.

Compression Techniques

Lossy Compression
  • Permanently removes some data
  • Higher compression ratios
  • Irreversible - original cannot be perfectly reconstructed
  • Examples: JPEG, MPEG, MP3
Lossless Compression
  • No data loss - original can be perfectly reconstructed
  • Lower compression ratios
  • Reversible compression
  • Examples: PNG, GIF, ZIP, TIFF (LZW)

1.3 Color Models and Color Space

RGB (Red-Green-Blue) Color Model

The RGB color model is an additive color model in which red, green, and blue light are added together in various ways to reproduce a broad array of colors.

  • Additive model: Colors are created by adding light
  • Device-dependent: Colors may appear differently on different devices
  • Primary colors: Red, Green, Blue
  • Black: (0,0,0) - White: (255,255,255) for 8-bit per channel

CMY/CMYK Color Model

The CMY (Cyan, Magenta, Yellow) model is a subtractive color model used in color printing. CMYK adds a key (black) component to improve dark colors and reduce ink usage.

  • Subtractive model: Colors are created by subtracting wavelengths from white light
  • Used in printing: CMYK is standard for color printing
  • Relationship to RGB: C = 1-R, M = 1-G, Y = 1-B (in normalized form)

HSV/HSI Color Spaces

HSV (Hue-Saturation-Value)
  • Hue: The color type (0°-360° on color wheel)
  • Saturation: The intensity/purity of color (0%-100%)
  • Value: The brightness of color (0%-100%)
  • More intuitive for humans than RGB
HSI (Hue-Saturation-Intensity)
  • Hue: The dominant wavelength (color)
  • Saturation: The purity of color
  • Intensity: The overall brightness
  • Similar to HSV but with different calculations

YCbCr Color Space (for compression)

YCbCr is a family of color spaces used in video and digital photography systems, particularly for compression (JPEG, MPEG).

Y (Luma)

Brightness component, carries most visual information

Cb (Chroma Blue)

Blue difference component, less important visually

Cr (Chroma Red)

Red difference component, less important visually

Chroma Subsampling: Since human vision is more sensitive to brightness than color, Cb and Cr components can be downsampled (4:2:2, 4:2:0) to reduce data without significant quality loss.

Color Space Conversions

Different color spaces are used for different purposes. Conversion formulas allow transformation between spaces:

Conversion Purpose Key Formula
RGB → HSV Color manipulation, filtering H = atan2(√3(G-B), 2R-G-B)
RGB → YCbCr Image compression (JPEG, MPEG) Y = 0.299R + 0.587G + 0.114B
RGB → CMYK Printing applications K = 1 - max(R,G,B)

Indexed Color vs. True Color

Indexed Color
  • Uses a color palette (color lookup table)
  • Each pixel stores an index to the palette
  • Limited colors (typically 256 or fewer)
  • Examples: GIF, 8-bit PNG
  • Smaller file size, faster processing
True Color
  • Direct color representation
  • Each pixel stores actual color values (e.g., RGB)
  • Millions of colors (24-bit: 16.7 million)
  • Examples: JPEG, 24-bit PNG, BMP
  • Larger file size, more realistic colors

1.4 Relationship Between Pixels

Adjacency

Adjacency defines which pixels are considered neighbors of a given pixel. Different adjacency types are used for different applications.

4-Connectivity
P

Only vertical and horizontal neighbors are considered adjacent (up, down, left, right).

8-Connectivity
P

All surrounding pixels are considered adjacent (including diagonals).

m-Connectivity (Mixed)
P

Modified 8-connectivity that prevents ambiguous paths, considering diagonal neighbors only when 4-connected neighbors are not both present.

Distance Metrics

Distance metrics measure the distance between two pixels with coordinates (x₁,y₁) and (x₂,y₂).

Distance Metric Formula Description Visualization
Euclidean √[(x₁-x₂)² + (y₁-y₂)²] Straight-line distance between two points
Manhattan (City Block) |x₁-x₂| + |y₁-y₂| Distance along grid lines (like city blocks)
Chebyshev (Chessboard) max(|x₁-x₂|, |y₁-y₂|) Maximum of horizontal and vertical distances

Connectivity and Labeling

Connected Components

A set of pixels where each pixel is connected to every other pixel in the set through a path of adjacent pixels.

Three connected components: Red region, Blue region, and Background

Labeling Algorithms

Process of assigning unique labels to connected components in a binary image.

  • Two-pass algorithm: Scan image twice to resolve labels
  • One-pass algorithm: Uses union-find data structure
  • Applications: Object counting, region analysis, blob detection

Region and Boundary Relationships

Region Properties
  • Area: Number of pixels in the region
  • Perimeter: Length of region boundary
  • Centroid: Center of mass of the region
  • Compactness: Ratio of area to perimeter squared
  • Orientation: Direction of major axis
Boundary Representations
  • Chain codes: Represent boundary as sequence of directions
  • Polygonal approximations: Represent boundary as polygon
  • Signatures: 1D representation of boundary shape
  • Fourier descriptors: Frequency domain representation

Boundary vs Region: Boundary-based representations are more compact but region-based representations are more robust to noise.
