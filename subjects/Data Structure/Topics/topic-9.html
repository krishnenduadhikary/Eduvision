Topic 9

Graphs

A graph is an abstract data structure used to implement the mathematical concept of graphs. It consists of a collection of **vertices** (or nodes) and **edges** that connect these vertices. Graphs are powerful tools for modeling situations where entities are related in pairs and can represent complex relationships beyond simple hierarchies.

A graph G is defined as an ordered set (V, E), where V(G) represents the set of vertices and E(G) represents the set of edges that connect these vertices.

Why are Graphs Useful?

  • Modeling family trees (parent-child relationships).
  • Representing transportation networks (airports, roads, shipping routes as nodes; flights, paths as edges).
  • Social networks, circuit designs, program flow, and much more.

13.2 Graph Terminology

Adjacent nodes/Neighbours: For an edge (u, v), nodes u and v are adjacent.
Degree of a node (deg(u)): Total number of edges connected to node u. An isolated node has degree 0.
Regular graph: A graph where each vertex has the same number of neighbours (same degree). A k-regular graph has vertices of degree k. (See Figure 13.3)
Path P = (v0, v1, ..., vk): A sequence of (k+1) nodes where vi-1 is adjacent to vi for i=1 to k. Length of path is k.
Closed path: A path where the start and end points are the same (v0 = vk).
Simple path: All nodes are distinct, except possibly v0 = vk. If v0 = vk, it's a closed simple path.
Cycle: A path in which the first and last vertices are the same. A simple cycle has no repeated edges or vertices (except first/last).
Connected graph: There is a path between any two vertices (u, v). A tree is a special connected graph with no cycles.
Complete graph (Kn): All its n nodes are fully connected (a path from one node to every other node). Has n(n-1)/2 edges.
Clique: A subset of vertices V' ⊆ V such that for every two vertices in V', there is an edge connecting them (i.e., a complete subgraph).
Labelled/Weighted graph: Edges are assigned data/weights (positive values representing cost/length). (See Figure 13.4(c))
Multiple edges: Distinct edges connecting the same end-points.
Loop: An edge with identical end-points, e = (u, u).
Multi-graph: A graph with multiple edges and/or loops. (See Figure 13.4(a))
Size of a graph: Total number of edges.

13.3 Directed Graphs (Digraphs)

A directed graph (digraph) is a graph where every edge has an assigned direction, represented as an ordered pair (u, v), meaning the edge starts at u (origin/initial) and ends at v (destination/terminal). Here, u is a predecessor of v, and v is a successor of u.

Terminology for Directed Graphs:
  • Out-degree (outdeg(u)): Number of edges originating at u.
  • In-degree (indeg(u)): Number of edges terminating at u.
  • Degree (deg(u)): indeg(u) + outdeg(u).
  • Source: Node with positive out-degree and zero in-degree.
  • Sink: Node with positive in-degree and zero out-degree.
  • Reachability: Node v is reachable from u if a directed path exists from u to v.
  • Strongly Connected: A path exists between every pair of nodes (u to v AND v to u).
  • Unilaterally Connected: For any pair of nodes u, v, there's a path from u to v OR from v to u (but not necessarily both).
  • Weakly Connected: Connected if edge directions are ignored. Nodes must have either out-degree or in-degree of at least 1.
  • Simple Directed Graph: No parallel edges, but may have cycles (loops are generally allowed if not more than one at a node).

13.5 Representation of Graphs

Common ways to store graphs in computer memory:

13.5.1 Adjacency Matrix Representation

An n x n matrix where n is the number of vertices. An entry aij = 1 if there's an edge from vertex vi to vj, and 0 otherwise. For weighted graphs, aij stores the weight (or infinity if no edge).

Conclusions from Adjacency Matrix:
  • For simple graphs (no loops), diagonal is all 0s.
  • Symmetric for undirected graphs.
  • Memory: O(n2).
  • Number of 1s (non-zero entries) equals number of edges (for unweighted).
  • Powers of Adjacency Matrix: Ak[i][j] gives the number of paths of length k from vi to vj.
Path Matrix (B* or P):

Pij = 1 if there is a path from vi to vj, 0 otherwise. Can be found by B* = A + A2 + ... + An (where addition is boolean OR, multiplication is boolean AND) or using Warshall's algorithm.

13.5.2 Adjacency List Representation

An array of linked lists. Each index `i` in the array corresponds to vertex `v_i`, and the linked list at `Adj[i]` stores all vertices adjacent to `v_i`. Preferred for sparse graphs. Adding new nodes/edges is generally easier than with matrices.

For directed graphs, sum of list lengths = total edges. For undirected, sum = 2 * total edges.

13.5.3 Adjacency Multi-list Representation

An edge-based rather than vertex-based representation. Consists of a directory of nodes and a set of linked lists storing edge information. Each edge appears in two adjacency lists (one for each end vertex). Useful for operations requiring easy access to edges.

13.6 Graph Traversal Algorithms

Examining nodes and edges systematically. Node states during traversal:

Status State of the node Description
1 Ready Initial state.
2 Waiting Node on queue/stack, waiting to be processed.
3 Processed Node has been completely processed.
13.6.1 Breadth-First Search (BFS)

Uses a queue. Explores starting node, then all its neighbours, then their unvisited neighbours, and so on, layer by layer.

Algorithm for Breadth-First Search (Figure 13.20)
  1. SET STATUS = 1 (ready state) for each node N.
  2. Enqueue the starting node A and set its STATUS = 2 (waiting).
  3. Repeat Steps 4 and 5 until QUEUE is empty:
  4. Dequeue a node N. Process it and set its STATUS = 3 (processed).
  5. Enqueue all neighbours of N that are in the ready state (STATUS = 1) and set their STATUS = 2 (waiting).
  6. EXIT.
Features of BFS:
  • **Space Complexity:** O(|V| + |E|) or O(bd) (b=branching factor, d=depth).
  • **Time Complexity:** O(|V| + |E|).
  • **Completeness:** Yes, finds a solution if one exists.
  • **Optimality:** Finds shortest path in terms of number of edges (for unweighted graphs).
  • **Applications:** Finding connected components, shortest path in unweighted graphs.
13.6.2 Depth-First Search (DFS)

Uses a stack. Expands starting node, then goes deeper along a path until goal or dead-end. Backtracks if dead-end or already visited node encountered.

Algorithm for Depth-First Search (Figure 13.22)
  1. SET STATUS = 1 (ready state) for each node in G.
  2. Push the starting node A on the stack and set its STATUS = 2 (waiting state).
  3. Repeat Steps 4 and 5 until STACK is empty:
  4. Pop the top node N. Process it and set its STATUS = 3 (processed state).
  5. Push on the stack all the neighbours of N that are in the ready state (STATUS = 1) and set their STATUS = 2 (waiting state).
  6. EXIT.
Features of DFS:
  • **Space Complexity:** O(|V|) or O(b*d) in worst case (lower than BFS usually).
  • **Time Complexity:** O(|V| + |E|).
  • **Completeness:** Yes (if graph is finite).
  • **Applications:** Finding paths, cycle detection, topological sorting, computing spanning trees.

13.8 Shortest Path Algorithms

13.8.1 Minimum Spanning Trees (MST)

A spanning tree of a connected, undirected graph G is a subgraph that is a tree and connects all vertices. An MST has a weight less than or equal to the weight of every other spanning tree. The total weight is the sum of its edge weights.

Applications of MSTs:
Connecting distances with minimum cost (networks, pipelines, roads).
Prim's Algorithm

A greedy algorithm that forms an MST by starting with an arbitrary vertex and iteratively adding the cheapest edge connecting a vertex in the MST to a vertex not yet in the MST, until all vertices are included.

Prim's Algorithm Steps (Figure 13.30)
  1. Select a starting vertex.
  2. Repeat Steps 3 and 4 until all vertices are in the MST:
  3. Select an edge (u,v) connecting the tree vertex u to a fringe vertex v that has minimum weight.
  4. Add the selected edge and vertex v to the minimum spanning tree T.
  5. EXIT.

Time Complexity: O(E log V) with a binary heap or O(V2) with adjacency matrix.

Kruskal's Algorithm

A greedy algorithm that finds an MST by adding edges in increasing order of weight, ensuring no cycles are formed. Uses a disjoint set data structure (Union-Find) to track connected components.

Kruskal's Algorithm Steps (Figure 13.33)
  1. Create a forest where each vertex is a separate tree.
  2. Create a priority queue Q containing all edges of the graph.
  3. Repeat Steps 4 and 5 while Q is NOT EMPTY:
  4. Remove an edge (u,v) from Q (edge with minimum weight).
  5. IF the edge obtained connects two different trees, then add it to the forest (MST). (Combine the two trees into one). ELSE Discard the edge.
  6. END.

Time Complexity: O(E log E) or O(E log V) mainly due to sorting edges and Union-Find operations.

13.8.4 Dijkstra's Algorithm

Finds the shortest path from a single source vertex to all other vertices in a weighted graph with non-negative edge weights. Widely used in network routing (IS-IS, OSPF).

Dijkstra's Algorithm (Figure 13.35)
  1. Select the initial node (source). Set its distance to 0, all others to infinity.
  2. Define an empty set N for visited nodes. Add source to N.
  3. Repeat until destination is in N or no more reachable nodes:
    1. For each neighbor of the newly added node not in N: Calculate new distance (current node's dist + edge weight). If less than current label, update.
    2. Pick node not in N with smallest temporary label, add to N. Make it current.

Time Complexity: O(V2) with adjacency matrix, O((V+E) log V) with adjacency list and binary heap.

Bellman-Ford Algorithm

Finds single-source shortest paths in a weighted digraph, capable of handling negative edge weights and detecting negative weight cycles.

How it works:

Calculates shortest paths in a bottom-up manner. After i-th iteration, shortest paths with at most i edges are found. Outer loop runs |V|-1 times. An additional iteration detects negative cycles if distances still improve.

Bellman-Ford Algorithm Steps
  1. Initialize distances: dist[src] = 0, all other dist[v] = ∞.
  2. Repeat |V|-1 times: For each edge (u,v) with weight w: RELAX(u,v,w) i.e., if dist[u] + w < dist[v], then dist[v]=dist[u] + w.
  3. For each edge (u,v): If dist[u] + w < dist[v], then graph contains a negative-weight cycle.

Time Complexity: O(VE).

13.8.5 Warshall's Algorithm (All-Pairs Shortest Path / Transitive Closure)

Determines if a path exists between all pairs of vertices. Generates the path matrix P (transitive closure) from the adjacency matrix A.

Warshall's Algorithm Steps (Figure 13.38)
  1. Initialize P with the adjacency matrix A (P0 = A). Add self-loops (P[i][i]=1).
  2. For k = 0 to n-1 (intermediate vertex):
  3. For i = 0 to n-1 (source vertex):
  4. For j = 0 to n-1 (destination vertex):
  5. Pk[i][j] = Pk-1[i][j] OR (Pk-1[i][k] AND Pk-1[k][j]).
  6. EXIT.

Time Complexity: O(V3).

13.9 Applications of Graphs

  • Circuit networks (vertices = connection points, edges = wires).
  • Transport networks (vertices = stations/airports, edges = routes).
  • Maps (vertices = cities/regions, edges = adjacency).
  • Program flow analysis (vertices = modules, edges = calls).
  • Project management (Activity Network Diagrams / PERT charts - Figure 13.43).
  • State transition diagrams (nodes = states, edges = transitions).
  • Social networks, web graphs, chemical compound structures.
