Introduction to Operating Systems

Understanding the fundamentals: What an OS is, its purpose, various types, core functions, and essential components.

1. Definition and Purpose of an OS

An Operating System (OS) is pivotal software that serves as an intermediary between the computer user and the computer hardware. It's responsible for managing the computer's hardware resources and providing an environment in which user programs can execute efficiently and conveniently.

Conceptual Diagram: OS as an Intermediary

User Applications ↔ Operating System ↔ Hardware (CPU, Memory, I/O, Disk)

Key Definitions:

  • "An operating system is software that manages the basic resources of a computer i.e Processor, Memory, Files, Peripheral devices, disk storage units etc..."
  • "An Operating System can be defined as an interface between user and hardware."

Primary Purposes of an OS:

  • Resource Management: Efficiently manages and allocates CPU time, memory space, file storage, and I/O devices. "OS performs the job of allocating resources for various tasks ... and reclaims the allocated resources after their use."
  • Execution Environment: "It provides an environment in which user can execute programs in a convenient and efficient manner."
  • Control Program: Oversees program execution, preventing errors and unauthorized use. "Purpose of OS is to transfer control automatically from one job to next job."
  • Abstraction Layer: Simplifies complex hardware interactions for users and applications.
  • Convenience & Efficiency: Makes the computer system user-friendly and optimizes resource utilization.

"The primary need for the OS arises from the fact that user needs to be provided with services and OS ought to facilitate the provisioning of these services."

2. Types of Operating Systems

Operating systems have evolved, resulting in diverse types tailored for specific applications and environments.

Batch Operating System

"The users of a batch operating system do not interact with the computer directly. Each user prepares his job on an off-line device like punch cards and submits it to the computer operator." Jobs with similar requirements are grouped into batches and processed sequentially by the OS to improve throughput.

Batch OS Workflow Diagram

Users submit Jobs → Operator batches Jobs → Computer processes Batches → Output

Problems with Batch Systems:
  • Lack of user interaction with the running job.
  • CPU often idle due to slow I/O devices.
  • Difficult to implement job priority.

Time-sharing Operating Systems (Multitasking)

"Time-sharing is a technique which enables many people, located at various terminals, to use a particular computer system at the same time. Time-sharing or multitasking is a logical extension of multiprogramming." The OS rapidly switches CPU allocation among multiple active jobs, giving each user the impression of dedicated system access.

Time-Sharing OS Concept Diagram

Central CPU ↔ (User 1 [Active], User 2 [Ready], User 3 [Active], ...)

Key Features:
  • Objective: Minimize user response time.
  • Mechanism: CPU scheduling and multiprogramming.
Advantages:
  • Quick response times for users.
  • Reduced software duplication.
  • Improved CPU utilization.
Disadvantages:
  • Reliability concerns.
  • Security and data integrity challenges for multiple users.
  • Complexity in data communication.

Distributed Operating System

"Distributed systems use multiple central processors to serve multiple real-time applications and multiple users. Data processing jobs are distributed among the processors accordingly." Manages a collection of independent, networked computers, making them appear as a single, coherent system. These are often loosely coupled systems.

Distributed OS Architecture Diagram

Communication Network connecting [Work Stations, Terminals, File Servers, DB Servers, Nodes]

Advantages:
  • Enhanced resource sharing across sites.
  • Faster data exchange (e.g., email).
  • Improved fault tolerance (if one site fails, others may continue).
  • Better service scalability and load balancing.

Real-Time Operating System (RTOS)

"A real-time system is defined as a data processing system in which the time interval required to process and respond to inputs is so small that it controls the environment." Crucial for applications with strict timing constraints, often embedded systems.

RTOS Concept

Sensors/Inputs → RTOS (Strict Timing) → Actuators/Outputs

Types:
  • Hard real-time systems: Guarantee task completion within deadlines (e.g., flight control, medical pacemakers). Often lack secondary storage and virtual memory.
  • Soft real-time systems: Prioritize critical tasks but can tolerate occasional deadline misses (e.g., multimedia streaming, online gaming).

Network Operating System (NOS)

"A Network Operating System runs on a server and provides the server the capability to manage data, users, groups, security, applications, and other networking functions." Facilitates resource sharing (files, printers) among computers on a network.

Network OS Diagram

Server (NOS, Storage) ↔ Network Hub ↔ [Client 1, Client 2, Client 3]

Examples: Windows Server, Linux (as a server), Novell NetWare.
Advantages: Centralized management, enhanced security, remote access.
Disadvantages: Higher cost, dependency on server, maintenance needs.

Mobile Operating System

"A mobile operating system (OS) is software that enables smartphones, tablets, and other mobile devices to run applications and programs." Manages mobile device hardware and provides a platform for mobile applications.

Mobile OS Ecosystem

Device Hardware ↔ Mobile OS ↔ (Applications, User Interface, Cloud Services)

Key Concepts: Function (hardware/software management), User Interface (touch, voice), Application Support (app stores), Hardware Management.
Prominent Examples: Android (Google), iOS (Apple).

3. Functions of an Operating System

The OS performs several vital functions to manage the computer system and provide a stable operating environment.

1. Process Management

"A program does nothing unless its instructions are executed by a CPU. A process is a program in execution."

The OS is responsible for activities related to process management such as:

  • Creating and deleting user and system processes.
  • Suspending and resuming processes.
  • Providing mechanisms for process synchronization.
  • Providing mechanisms for process communication.
  • Handling deadlocks.

    2. Main Memory Management

    "Main memory is central to the operation of a modern computer system... Main memory stores the quickly accessible data shared by the CPU & I/O device."

    The OS manages this critical resource by:

    • Tracking memory usage (which parts are in use and by whom).
    • Deciding which processes/data to move into and out of memory.
    • Allocating and deallocating memory space as needed.

    3. File Management

    "File management is one of the most important components of an OS... The OS provides a uniform logical view of information storage."

    The OS handles files by:

    • Creating and deleting files and directories.
    • Providing primitives for file/directory manipulation.
    • Mapping files onto secondary storage.
    • Supporting file backup.

    4. I/O System Management

    "One of the purposes of an OS is to hide the peculiarities of specific hardware devices from the user."

    The I/O subsystem includes a memory management component (buffering, caching, spooling), a general device-driver interface, and specific device drivers to manage I/O operations efficiently and abstract hardware details.

    5. Protection or Security

    "If a computer system has multi users & allow the concurrent execution of multiple processes then the various processes must be protected from one another's activities."

    The OS enforces security policies to control access to resources, ensuring that only authorized processes can operate on files, memory segments, CPU, etc. This involves mechanisms for authentication, authorization, and data protection.

    6. Command Interpretation

    "One of the most important functions of the OS is connected interpretation where it acts as the interface between the user & the OS."

    The command interpreter (e.g., shell or GUI) processes user commands and translates them into system calls for the kernel to execute, enabling user interaction with the system.

    4. System Components

    "In an operating system, the kernel acts as the central interface between user applications and the underlying hardware, while the shell provides a user interface to interact with the kernel."

    Linux Operating System Architecture (Conceptual)
    User ↔ Applications ↔ Shell ↔ Kernel ↔ Hardware

    1. Hardware Interface

    Definition: "The hardware interface is the connection between the operating system and the physical components of the computer."

    Function: Allows OS to access and control hardware resources like the CPU, memory, disk drives, and network interfaces.

    Mechanism: This is typically achieved through device drivers, which provide a standardized interface for interacting with diverse hardware.

    2. Kernel

    Definition: "The kernel is the core of the operating system, responsible for managing system resources and interacting with the hardware interface." It is the first part of the OS to load into memory during boot-up and remains resident.

    Function: It handles essential tasks such as process scheduling, memory management, file system management, and device driver management.

    Role: It provides an abstraction layer between user applications and the hardware, shielding applications from the complexities of direct hardware control and ensuring system stability.

    3. Shell

    Definition: "The shell is a user interface that allows users to interact with the operating system, typically through a command-line interface (CLI) or a graphical user interface (GUI)."

    Function: "It interprets user commands and passes them to the kernel, which then executes the requested actions." It effectively translates human-readable commands into operations the kernel can understand.

    Example: "A common example of a shell is the Bash shell used in Linux systems." Other examples include PowerShell (Windows), Zsh (macOS/Linux), and graphical shells like Windows Explorer or GNOME Shell.

    5. System Calls & OS Structure

    5.1 System Calls

    "System calls provide the interface between a process & the OS. These are usually available in the form of assembly language instruction," though modern systems often provide C library wrappers.

    "Some systems allow system calls to be made directly from a high level language program like C, BCPL and PERL etc."

    System calls are how user programs request services from the operating system kernel. They transition the CPU from user mode to kernel mode to perform privileged operations.

    Major Categories of System Calls:

    • Process Management and Control: Calls to create, terminate, load, execute, and manage processes (e.g., `fork()`, `exec()`, `exit()`, `wait()`). Also includes managing process attributes and memory allocation for processes.
    • File Management/File Manipulation/File Handling: Calls for creating, deleting, opening, closing, reading, writing, and repositioning files, as well as managing file attributes (e.g., `open()`, `read()`, `write()`, `close()`, `unlink()`).
    • Device Management: Calls to request, release, read from, write to, and reposition devices. Often involves I/O control operations (e.g., `ioctl()`).
    • Information Maintenance/Management: Calls to get or set system information like time/date, system data (OS version, user count), and process/file/device attributes.
    • Communication Management: Calls for creating/deleting communication connections, sending/receiving messages between processes (Inter-Process Communication - IPC), and managing remote devices.
      Modes of Communication (Details)

      Message passing model: Information is exchanged through an OS-provided facility. Processes have IDs and explicitly send/receive messages. Requires cooperation and synchronization.

      Shared memory model: Processes map a common region of memory into their address spaces. Communication occurs by reading/writing to this shared area. Faster but requires careful synchronization to avoid conflicts.

    5.2 Operating System Structures

    "The design of operating system architecture traditionally follows the separation of concerns principle," aiming for manageability and clarity.

    1. Simple Structure (e.g., MS-DOS)

    "There are several commercial system that don't have a well-defined structure... MS-DOS is an example of such system. It was not divided into modules carefully." Applications could often bypass OS layers and directly access hardware (like BIOS). "There was no CPU Execution Mode (user and kernel), so errors in applications could cause the whole system to crash."

    MS-DOS Structure Diagram
    Application Program ↔ Resident System Program ↔ MS-DOS Device Drivers ↔ ROM BIOS Device Drivers

    2. Monolithic Approach

    "A monolithic design of the operating system architecture makes no special accommodation for the special nature of the operating system." The entire OS is a single, large program running in kernel mode ("maximum privileges"). All components are tightly coupled. "CP/M and DOS are simple examples of monolithic operating systems."

    Advantages: Low communication overhead between components, potentially high performance.

    Disadvantages: Difficult to develop, debug, maintain, and extend. A bug in one part can crash the entire system.

    3. Layered Approach

    "In the layered approach, the OS is broken into a number of layers (levels) each built on top of lower layers. The bottom layer (layer 0) is the hardware & top most layer (layer N) is the user interface." Each layer provides services only to the layer above it and uses services only from the layer below it.

    "The main advantage of the layered approach is modularity." This simplifies design, implementation, debugging, and system verification, as layers can be developed and tested independently.

    Layered OS Structure Example
    Layer 5: User Program
    Layer 4: I/O Management
    Layer 3: Operator Process Communication
    Layer 2: Memory Management
    Layer 1: CPU Scheduling
    Layer 0: Hardware

    Disadvantages: Can be less efficient due to overhead of calls passing through multiple layers. Defining layers appropriately can be difficult.

    4. Microkernel Approach

    "This structures the operating system by removing all nonessential portions of the kernel and implementing them as system and user level programs." The microkernel itself is minimal, typically providing only basic process and memory management, and an inter-process communication (IPC) facility.

    Other OS services (file systems, device drivers, network stacks) run as separate processes in user space. "Communication between components of the OS is provided by message passing" via the microkernel.

    Microkernel Architecture
    User Mode: [App, File System, Device Driver] ↔ Messages ↔ Kernel Mode: [Microkernel (IPC, Mem Mgmt, CPU Sched)] ↔ Hardware

    Benefits: Improved reliability and security (failure in a user-space service doesn't crash kernel), easier to extend and port.

    Disadvantage: "Poor performance due to increased system overhead from message passing."

    5. Modular Approach (Hybrid)

    Most modern OS (like Linux, Windows, macOS) use a modular approach, which is a hybrid. They have a relatively large monolithic kernel but also support dynamically loadable modules (e.g., for device drivers, file systems, network protocols). These modules can be loaded and unloaded at runtime without rebooting.

    Advantages: Combines performance of monolithic kernels (modules run in kernel space) with flexibility and maintainability (modules are somewhat independent).
