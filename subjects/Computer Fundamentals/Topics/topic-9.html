Computer Networks & Internet

Exploring how computers connect, share information, and access global resources through networks, the internet, and cloud services.

Computer Network

A network is a set of devices (often referred to as nodes) connected by communication links. A node can be a computer, printer, or any other device capable of sending and/or receiving data generated by other nodes on the network.

Software modules in one system are used to communicate with one or more software modules in the distance System. Such interfaces across a distance are termed as “peer-to-peer” interfaces; and the local interfaces are termed as “service” interfaces. The modules on each end are organized as a sequence of functions called “layers”. The set of modules organized as layers is also commonly called a “protocol stack”.

Over the years, some layered models have been standardized. The ISO Open Systems Interconnection (ISO/OSI) layered model has seven layers and was developed by a set of committees under the auspices of International Standards Organization (ISO).

Classification of computer networks:

Based on geographical area:

  • Local Area Networks (LANs)
  • Wide Area Networks (WANs)
  • Metropolitan Area Networks (MANs)

Local Area Network (LAN)

LAN is a group of computers, as shown in Fig. 14.4, that are connected in a small area such as building, home, etc. Through this type of network, users can easily communicate with each other by sending and receiving messages. LAN is generally used for connecting two or more personal computers through some medium such as twisted pair, coaxial cable etc. Though the number of computers connected in a LAN is limited, the data is transferred at an extremely faster rate.

LAN Diagram

Wide Area Network (WAN)

WAN is a group of computers that are connected in a large area such as continent, country, etc. WAN is generally used for connecting two or more LANs through some medium such as leased telephone lines, microwaves, etc. In WAN, data is transferred at slow rate. A typical WAN network is shown in Fig. 14.5.

WAN Diagram

Metropolitan Area Network (MAN)

MAN is a network of computers that covers a large area like a city. The size of the MAN generally lies between that of LAN and WAN, typically covering a distance of 5 Km to 50 Km. The geographical area covered by MAN is comparatively larger than LAN but smaller than WAN. MAN is generally owned by private organisations. MAN is generally connected with the help of optical fibres, copper wires etc. One of the most common example of MAN is cable television network within a city as shown in Fig. 14.6. A network device known as router is used to connect the LANs together. The router directs the information packets to the desired destination.

Network Devices

Cables transmit data as signals. A concentrator (e.g., hub, switch) amplifies signals and provides an interface to connect multiple devices. Repeater, hub, switch, bridge, and gateway are examples. Devices like Bridge, Router, and Gateway are used to interconnect LANs, especially if they use different protocols.

Router

Used to connect heterogeneous networks (different technologies, media, addressing). Can connect LANs, a LAN and a WAN, or two WANs. Operates at OSI Layer 3 (Network Layer). Determines the shortest/fastest path and routes packets based on IP addresses.

Router Diagram

Switch

Connects multiple computers or network segments. Smarter than a hub; a switch creates a direct connection (simulated LAN segment) between the source and destination. Works at OSI Layer 2 (Data Link Layer), using MAC addresses to forward frames. Can extend network distance and filter traffic.

Switch Diagram

Modem (Modulator-Demodulator)

Converts digital data to analog signals for transmission (modulation) and vice-versa (demodulation). Enables communication over various media.

Hub

Like a repeater with multiple ports. Does not amplify signals. Operates at OSI Layer 1 (Physical Layer), treating data as signals. Connects multiple segments of the same network, often in a star topology. Broadcasts data to all connected devices, increasing traffic. Best for small LANs (8-10 computers).

Internet Basics

Internet Overview

The Internet is a giant network of networks. It may include PCs, servers, printers, etc. Connected through communication channels. Early research by US Department of Defense (1962) led to ARPAnet (Advanced Research Project Agency) to connect US Defense Department network.

What did the Internet come from?

  • Original aim: Allow university research computers to 'talk' to each other.
  • ARPAnet's design allowed messages to be rerouted, ensuring network function even if parts were destroyed.
  • Users of the Internet took a direction of their own.

History of the Internet:

  • 1965: First long-distance communication (MIT and California).
  • 1969: Four computers connected via ARPAnet.

IP Addressing

An Internet Protocol (IP) Address is a numerical label assigned to each device connected to a computer network that uses the Internet Protocol for communication. It serves two principal functions: host or network interface identification and location addressing. An IP Address is a 32-bit unique address (for IPv4). Always consists of 4 numbers separated by periods, with numbers ranging from 0 to 255. Example: 192.168.12.1

Format of an IP address:

  • Dotted decimal notation
  • Hexadecimal notation (less common for IPv4 display)

The IP address (32 bits for IPv4) is divided into network part (Network ID) and host part (Host ID). Each octet (8 bits) is converted to decimal. The format of the IP address differs with the class (A, B, C, D, E).

IP Octet Diagram

Classes of networks (IPv4):

  • Class A: 1-126 in first octet. First 8 bits network, next 24 hosts. For large networks. Range: 1.0.0.1 - 126.255.255.254.
  • Class B: 128-191 in first octet. First 16 bits network, next 16 hosts. For medium networks. Range: 128.1.0.1 - 191.255.255.254.
  • Class C: 192-223 in first octet. First 24 bits network, next 8 hosts. For small networks. Range: 192.0.1.1 - 223.255.254.254.
  • Class D: For multicasting. 224-239 in first octet. Range: 224.0.0.0 - 239.255.255.255.
  • Class E: Reserved for future use/experimenting. 240-255 in first octet. Range: 240.0.0.0 - 255.255.255.254.
IP Class Structure Diagram

Types of IP Addresses: IPv4 and IPv6

  • IPv4 Addresses: 32-bit. Limited to ~4.29 billion unique addresses. Example: 12.244.233.165.
  • IPv6 Addresses: 128-bit. Accommodates many more addresses. Example: 2001:0db8:0000:0000:0000:ff00:0042:7879. Can coexist with IPv4 (Dual Stack).

Private (Local) IP Address vs Public (Global) IP address

Private IPs are for internal networks (LANs), not routable on the internet. Public IPs are provided by ISPs, globally unique, and routable. NAT (Network Address Translation) translates private IPs to public IPs for internet access.

Subnet mask

A 32-bit combination helping identify which part of an IP address is network and which is host. Used by TCP/IP to determine if a host is local or remote. Example: 255.0.0.0 for Class A default.

Media Access Control (MAC) / Physical Address

A 48-bit address uniquely coded into a device during manufacturing (identifies network adapter). Used for communication on the same LAN. Not sent to gateway. If a machine needs to communicate outside its network, an IP address is required.

Default gateways

A router specified on a host that links the host's subnet to other networks.

Domain Name System (DNS)

Human-readable website addresses (e.g., google.com). DNS is like the internet's phonebook, translating domain names to numeric IP addresses. Also provides mail server lists. Hierarchical system for domain names (e.g., .com, .gov, .uk).

DNS Servers

Keep files (zone files) mapping domain names to IPs. Provided by ISPs or can be specific servers. Your computer queries DNS servers to resolve domain names to IPs for connection.

How DNS Servers speed up search

Caching DNS records to avoid repeated lookups (cached DNS record).

Uniform Resource Locator (URL)

Standard address to find a webpage, webserver, or other internet devices. Typical format: `http://www.payne-gallway.co.uk/computing/student_resources.html` (Protocol | Host | Domain Name | Folder | Document Name).

World Wide Web (WWW or Web)

An information space where documents (URLs) are interlinked by hypertext, accessible via the Internet. Invented by Tim Berners-Lee (1989). Web pages are primarily text documents formatted with HTML. May contain images, video, audio, software components. Hyperlinks permit navigation. Client/server computing model.

WWW Concept Image

Browsers

Program to access the WWW (e.g., Chrome, Firefox, Safari, Internet Explorer). "Browses" the WWW, requests documents. Can be graphical or text-only.

Search Engines

Programs that search documents for keywords and return a list. Often used for World Wide Web. They search a database of web pages harvested and cached by "spiders" or "crawlers".

How do Search Engines Work?

Spiders crawl the web, finding pages by following links. An "indexing" program identifies text, links, and other content, storing it for fast searching.

Cloud Computing

Introduction

Cloud computing relies on shared computing resources rather than local servers or personal devices to handle applications.

Definition by NIST:

"A model for enabling ubiquitous, convenient, on-demand network access to a shared pool of configurable computing resources (e.g., networks, servers, storage, applications and services) that can be rapidly provisioned and released with minimal management effort or service provider interaction."

  • Ability / space where you store your data, process it and can access anywhere from the world.
  • As a metaphor for the internet.
  • Cloud computing is: Storing/Processing Data/Applications on remote servers, Accessing Data/Applications via internet.

What is a cloud service?

Taking services and moving them outside an organization's firewall. Delivered over the Web, paid for on an as-needed or pay-per-use basis. Service: Reusable, fine-grained components across a vendor's network. (IaaS, PaaS, SaaS, DaaS, NaaS, CaaS are some services).

1.1 Characteristics (OR) Features of Cloud Environments (NIST):

  1. On-demand self-service: Customers can sign up, pay for, and start using resources without help from a sales agent.
  2. Broad network access: Access cloud services via the Internet.
  3. Resource pooling: Many customers (individuals, organizations) use the same servers, storage, or computing resources.
  4. Rapid elasticity or expansion: Customers can easily scale their use of resources up or down.
  5. Measured service: Pay for the amount of resources used in a given period.

1.2 Applications:

  • i) Storage: Cloud keeps many copies of storage. Using these copies, it extracts another resource if one fails.
  • ii) Database: Repositories for information with links, helping make data searchable.

Advantages (of cloud databases/storage):

  • Improved availability
  • Improved performance
  • Price (less expensive than network of smaller computers)
  • Flexibility

Disadvantages (of cloud databases/storage):

  • Complexity
  • Labor costs
  • Security (data fragments, site housing)
  • Integrity

Types of Clouds

  1. Public Cloud: Established by any subscriber with an internet connection, access to the cloud space.
  2. Private Cloud: Established for a specific group or organization, limits access.
  3. Community Cloud: Shared among two or more organizations that have similar cloud requirements.
  4. Hybrid Cloud: Essentially a combination of at least two clouds, where the clouds included are a mixture of public, private, or community.

Choosing a Cloud Provider & Service Types

Consider control vs. convenience. Three main types of cloud services:

  • Software as a Service (SaaS): Provider gives access to both resources and applications. Makes it unnecessary to have a physical copy of software on your devices. Least control for user.
  • Platform as a Service (PaaS): Provider gives access to components required to develop and operate applications over the internet. More control than SaaS.
  • Infrastructure as a Service (IaaS): Subscriber completely outsources storage and resources, such as hardware and software. Most control for user (over OS, storage, deployed apps).

Security

Information in the cloud is often valuable. Potentially secure data stored on shared computers. Understand provider's security measures (encryption, protection methods, firewalls, backups). Consider negotiation terms for small businesses. Giving up direct control of data requires trust in provider's expertise.
