    Introduction to AI Problem Solving

    Problem solving has been the key area of concern for Artificial Intelligence. It is a process of generating solutions from observed data or given conditions. It's not always possible to use direct methods; often, AI needs to use indirect or model-based methods, which typically involve search.

    To solve a problem of building a system, you should take the following steps:

    1. Define the problem accurately including detailed specifications and what constitutes a suitable solution.
    2. Scrutinize the problem carefully, for some features may have a central affect on the chosen method of solution.
    3. Segregate and represent the background knowledge needed in the solution of the problem.
    4. Choose the best solving techniques for the problem to solve it.

    A 'problem' is characterized by a set of goals, a set of objects, and a set of operations. These could be ill-defined and may evolve during problem solving.

    A 'problem space' is an abstract space which encompasses all valid states that can be generated by the application of any combination of operators on any combination of objects. A solution is a combination of operations and objects that achieve the goals.

    A 'search' refers to the search for a solution in a problem space. Search proceeds with different types of 'search control strategies'.

    General Problem Solver (GPS)

    The General Problem Solver (GPS) was a computer program created in 1957 by Simon and Newell to build a universal problem solver machine. GPS was based on Simon and Newell's theoretical work on logic machines. GPS in principle can solve any formalized symbolic problem, such as theorems proof and geometric problems and chess playing. GPS solved many simple problems, such as the Towers of Hanoi, that could be sufficiently formalized, but GPS could not solve any real-world problems.

    Formulating Problems for AI (State Space Search)

    To build a system to solve a particular problem, we need to define the problem precisely. This involves finding input situations as well as final situations for an acceptable solution to the problem.

    Problem definitions require:

    1. A state space that contains all possible configurations of relevant objects.
    2. One or more initial states from which the problem-solving process may start.
    3. One or more goal states that would be acceptable solutions.
    4. A set of rules (operators) that describe the actions available.

    The problem can then be solved by using the rules, in combination with an appropriate control strategy, to move through the problem space until a path from an initial state to a goal state is found. This process is known as 'search'.

    Graph vs. Tree Representation

    Image: Diagram showing a generic graph and a tree structure (Fig 2.1 from data)

    A problem space is represented by a directed graph, where nodes represent search states and paths represent the operators applied to change the state. Trees are often used to simplify search algorithms as they decrease complexity, though some nodes might be duplicated.

    Example: The Water Jug Problem

    We use two jugs called four and three; four holds a maximum of four gallons of water and three a maximum of three gallons of water. How can we get two gallons of water in the four-gallon jug? The state space is a set of ordered pairs (x, y) where x is water in the four-gallon jug and y is water in the three-gallon jug.

    Water Jug Problem Details (Initial State, Goal State, Rules)

    Initial State: (0, 0)

    Goal State: (2, n) where n can be any value.

    Production Rules: (Examples from Fig 2.2)

    1. (x, y) if x < 4 → (4, y)   {Fill 4-gallon jug}
    2. (x, y) if y < 3 → (x, 3)   {Fill 3-gallon jug}
    3. (x, y) if x > 0 → (0, y)   {Empty 4-gallon jug}
    4. (x, y) if y > 0 → (x, 0)   {Empty 3-gallon jug}
    5. (x, y) if x+y ≥ 3 and y < 3 → (x-(3-y), 3)   {Pour from 4 to 3 until 3 is full}
    6. ... and other rules for pouring between jugs and emptying.

    Uninformed (Blind) Search Algorithms

    These algorithms explore the search space without any domain-specific knowledge about the problem, other than the problem definition itself. They do not use heuristics.

    1. Breadth-First Search (BFS)

    BFS explores all nodes at the present depth level before moving on to nodes at the next depth level. It uses a FIFO queue.

    • Completeness: Yes (if the branching factor is finite).
    • Optimality: Yes (if all step costs are equal).
    • Time Complexity: O(bd) where b is branching factor, d is depth of solution.
    • Space Complexity: O(bd) (stores all explored nodes).
    Image: BFS tree traversal example and graph example.
    2. Depth-First Search (DFS)

    DFS explores as far down a branch as possible before backtracking. It uses a LIFO stack.

    • Completeness: No (can get stuck in infinite loops on infinite graphs without cycle detection or depth limits). Yes for finite graphs.
    • Optimality: No (may find a suboptimal solution first).
    • Time Complexity: O(bm) where m is maximum depth of search space.
    • Space Complexity: O(bm) (stores only the current path and unexplored siblings).
    Image: DFS tree traversal example and graph example.
    3. Uniform Cost Search (UCS)

    UCS expands the node with the least path cost (g(n)) from the start node. It uses a priority queue. It is optimal when step costs are non-negative.

    • Completeness: Yes (if step costs > ε > 0).
    • Optimality: Yes.
    • Time Complexity: O(b1+C*/ε) where C\* is cost of optimal solution.
    • Space Complexity: O(b1+C*/ε).
    Image: UCS graph example.
    4. Depth-Limited Search (DLS)

    DLS is DFS with a predetermined depth limit `l`. Solves the infinite path problem of DFS but introduces incompleteness if the shallowest goal is beyond the limit.

    5. Iterative Deepening Search (IDS) / Iterative Deepening DFS (IDDFS)

    IDS combines benefits of DFS (space complexity) and BFS (completeness, optimality for unit costs). It performs DLS repeatedly with increasing depth limits (0, 1, 2, ...). While it re-generates nodes, it's often preferred when search space is large and depth of solution is unknown.

    Informed (Heuristic) Search Algorithms

    These algorithms use domain-specific knowledge (heuristics) to find solutions more efficiently. A heuristic function, h(n), estimates the cost from node n to the nearest goal.

    What is a Heuristic?

    A heuristic is a technique designed for solving a problem more quickly when classic methods are too slow, or for finding an approximate solution when classic methods fail to find any exact solution. This is achieved by trading optimality, completeness, accuracy, or precision for speed. Heuristics are rules of thumb, educated guesses, intuitive judgments, or simply common sense.

    Heuristic Search Techniques Overview

    Image: Hierarchical tree of search algorithms showing No Heuristics vs. Informed Search branches (Fig. Different Search Algorithms from data).
    1. Greedy Best-First Search

    Expands the node that appears to be closest to the goal, based solely on the heuristic function h(n). It's "greedy" because it tries to get to the goal as fast as possible without considering the cost already incurred.

    • Completeness: No (can get stuck in loops or go down an infinite path).
    • Optimality: No.
    • Time Complexity: O(bm) in worst case, but a good heuristic can reduce this significantly.
    • Space Complexity: O(bm) in worst case.
    2. A* Search

    A\* search is one of the most widely used best-first search algorithms. It evaluates nodes by combining g(n) (the cost to reach the node) and h(n) (the estimated cost from the node to the goal): f(n) = g(n) + h(n).

    A\* Search Formula

    Image: f(n) = g(n) + h(n) breakdown diagram.
    • Completeness: Yes (if branching factor is finite and step costs > ε > 0).
    • Optimality: Yes, if the heuristic h(n) is admissible (never overestimates the true cost to the goal) and, for graph search, consistent (monotonicity: h(n) ≤ cost(n, n') + h(n')).
    • Time Complexity: Exponential in worst case, but highly dependent on the quality of the heuristic.
    • Space Complexity: Keeps all generated nodes in memory, O(bd).

    A\* Search Example

    3. AO* Search (And-Or Graph Search)

    AO* search is used for problems that can be decomposed into subproblems, where some subproblems (AND nodes) must all be solved, or one of several alternative subproblems (OR nodes) must be solved. This is common in problem reduction scenarios.

    It uses heuristic estimates and revises them as the search progresses, trying to find the most promising solution tree.

    Local Search Algorithms

    Local search algorithms operate using a single current node (rather than multiple paths) and generally move only to neighbors of that state. They are not systematic but can be very efficient. They are often used for optimization problems.

    How does AI-driven Local Search work?

    1. Initialization: Start with an initial solution.
    2. Evaluation: Evaluate the quality of the solution using an objective function.
    3. Neighborhood search: Generate neighboring solutions by small modifications.
    4. Selection: Choose the best neighboring solution.
    5. Termination: Stop when a criterion is met (e.g., no improvement, max iterations).
    1. Hill Climbing

    Hill climbing is a "greedy" local search algorithm that continuously moves in the direction of increasing value (uphill) to find the peak or best state. It terminates when it reaches a peak where no neighbor has a higher value.

    • Simple Hill Climbing: Selects the first better neighbor.
    • Steepest-Ascent Hill Climbing: Examines all neighbors and picks the one with the greatest improvement.
    • Stochastic Hill Climbing: Chooses a better neighbor at random.

    Problems with Hill Climbing:

    • Local Maxima: Peaks that are not the global maximum.
    • Plateaus: Flat areas where no neighbor is better.
    • Ridges: Areas where movement in any single direction is downhill, but the ridge itself slopes upwards.

    2. Simulated Annealing

    Simulated annealing is a probabilistic technique for approximating the global optimum of a given function. It allows "bad" moves (downhill) occasionally, with decreasing probability as the search progresses (controlled by a "temperature" parameter). This helps escape local optima.

    Basic Steps:

    1. Start with an initial solution and high temperature.
    2. Generate a new neighboring solution.
    3. If better, accept it.
    4. If worse, accept it with a probability that depends on the temperature and how much worse it is.
    5. Decrease temperature.
    6. Repeat until a stopping criterion is met.

    Constraint Satisfaction Problems (CSP)

    CSPs involve finding solutions that satisfy a set of constraints. A state is defined by an assignment of values to a set of variables, and a goal test is a set of constraints that the values must obey.

    Key Components:

    • Variables (X): A set of variables {X1, ..., Xn}.
    • Domains (D): A set of domains {D1, ..., Dn}, one for each variable.
    • Constraints (C): A set of constraints specifying allowable combinations of values.

    Techniques for Solving CSPs:

    • Backtracking Search: A depth-first search that assigns values to variables one by one and backtracks when a variable has no legal values left to assign or a constraint is violated.
    • Forward Checking: When a variable X is assigned a value, check each unassigned variable Y that is connected to X by a constraint and delete from Y's domain any value inconsistent with the value chosen for X.
    • Constraint Propagation (e.g., Arc Consistency): Enforcing constraints across variables to reduce the search space. For example, for an arc X → Y, X is arc-consistent with respect to Y if for every value in DX, there is some value in DY that is consistent.
    Example: Cryptarithmetic (SEND + MORE = MONEY)

    Assign unique digits (0-9) to letters S, E, N, D, M, O, R, Y such that the addition is correct. Constraints: M ≠ 0, all letters have unique digits.

    Problem Characteristics & Production Systems

    Problem Characteristics for Analysis

    To use heuristic search effectively, analyze the problem for characteristics like:

    • Decomposability into independent subproblems.
    • Possibility of undoing solution steps (recoverable vs. irrecoverable).
    • Predictability of the problem universe (certain vs. uncertain outcomes).
    • Nature of solution (state vs. path).
    • Role of knowledge.

    Production Systems

    Production systems provide a structure for performing search. They consist of:

    1. A set of rules (IF-THEN or condition-action pairs).
    2. A database of current facts (working memory).
    3. A control strategy that specifies rule selection and conflict resolution.
    4. A rule firing module.

    Search Algorithm Performance and Complexity

    When evaluating search algorithms, we consider:

    • Completeness: Does it always find a solution if one exists?
    • Optimality: Does it always find the least-cost solution?
    • Time Complexity: How long does it take as a function of problem size?
    • Space Complexity: How much memory does it need?

    Big-O Notation:

    Big-O notation describes the limiting behavior of a function when the argument tends towards a particular value or infinity. It's used to classify algorithms according to their running time or space requirements as the input size grows.

    • O(1): Constant time/space.
    • O(log n): Logarithmic.
    • O(n): Linear.
    • O(n log n): Log-linear.
    • O(n2): Quadratic.
    • O(2n): Exponential.
    • O(n!): Factorial.
