    Introduction to Knowledge Representation (KR)

    For the purpose of solving complex problems encountered in AI, we need both a large amount of knowledge and some mechanism for manipulating that knowledge to create solutions to new problems. A variety of ways of representing knowledge (facts) have been exploited in AI programs. In all variety of knowledge representations, we deal with two kinds of entities:

    • A. Facts: Truths in some relevant world. These are the things we want to represent.
    • B. Representations of facts in some chosen formalism: These are things we will actually be able to manipulate.

    One way to think of structuring these entities is at two levels:

    1. The knowledge level, at which facts are described.
    2. The symbol level, at which representations of objects at the knowledge level are defined in terms of symbols that can be manipulated by programs.

    The facts and representations are linked with two-way mappings called representation mappings. The forward representation mapping maps from facts to representations. The backward representation mapping goes the other way, from representations to facts.

    Framework of Knowledge Representation

    The Knowledge Representation models/mechanisms are often based on: Logic, Rules, Frames, Semantic Net.

    Knowledge Representation Framework

    Image: Problem → (Solve/Represent) Representation → (Interpret/Compute) Output → (Informal/Formal) Solution.

    The problem solving requires:

    • Formal knowledge representation.
    • Conversion of informal knowledge to a formal knowledge that is the conversion of implicit knowledge to explicit knowledge.

    A good knowledge representation system should have following properties:

    1. Representational Adequacy: The ability to represent all kinds of knowledge that are needed in that domain.
    2. Inferential Adequacy: The ability to manipulate the representational structures to derive new structures corresponding to new knowledge inferred from old.
    3. Inferential Efficiency: The ability to incorporate additional information into the knowledge structure that can be used to focus the attention of the inference mechanisms in the most promising direction.
    4. Acquisitional Efficiency: The ability to acquire new knowledge using automatic methods wherever possible rather than reliance on human intervention.

    Types of Knowledge

    Knowledge is categorized into two major types:

    1. Tacit Knowledge (Informal/Implicit):
      • Exists within a human being; it is embodied.
      • Difficult to articulate formally.
      • Difficult to communicate or share.
      • Hard to steal or copy.
      • Drawn from experience, action, subjective insight.
    2. Explicit Knowledge (Formal):
      • Exists outside a human being; it is embedded.
      • Can be articulated formally.
      • Can be shared, copied, processed and stored.
      • Easy to steal or copy.
      • Drawn from the artifact of some type as a principle, procedure, process, concepts.

    Knowledge Representation Schemes

    1. Relational Knowledge

    The simplest way to represent declarative facts is a set of relations of the same sort used in the database system. It provides a framework to compare two objects based on equivalent attributes. The table below shows a simple way to store facts.

    Player Height Weight Bats - Throws
    Aaron 6-0 180 Right - Right
    Mays 5-10 170 Right - Right
    Ruth 6-2 215 Left - Left
    Williams 6-3 205 Left - Right

    This representation provides little opportunity for inference (e.g., "Who is the heaviest player?" requires a procedure).

    2. Inheritable Knowledge

    Knowledge elements inherit attributes from their parents. This is common in design hierarchies (functional, physical domains). The basic KR needs to augment with inference mechanisms. Property inheritance: objects or elements of specific classes inherit attributes and values from more general classes.

    Inheritance Hierarchy Example

    Image: Adult Male → Musician (isa) → (Jazz / Avant Garde Jazz) → Miles Davis / John Zorn (instance).

    Boxed nodes are objects/values. Arrows point from object to its value. This is a slot and filler structure, semantic network, or collection of frames.

    3. Inferential Knowledge

    This knowledge generates new information from the given information, without further data gathering. Example: Predicate logic can be used to infer other values or relations. Representing knowledge as formal logic: "All dogs have tails" → ∀x: dog(x) → has_tail(x).

    Advantages: A set of strict rules, can derive more facts, truths of new statements can be verified, guaranteed correctness.

    4. Procedural Knowledge

    A representation in which the control information, to use the knowledge, is embedded in the knowledge itself. Example: computer programs, directions, and recipes. Knowledge is encoded in procedures that know how to do specific things.

    Advantages: Can represent heuristic or domain-specific knowledge, extended logical inferences like default reasoning. Side effects of actions may model.

    Disadvantages: Completeness (not all cases may represent), consistency (not all deductions may be correct), modularity sacrificed, changes in knowledge base might have far-reaching effects, cumbersome control information.

    Using Logic for Knowledge Representation

    Propositional Logic

    Propositional logic is useful because it is simple to deal with and a decision procedure for it exists. Facts are represented as simple propositions. Example: "Marcus is a man." (Man_Marcus). However, it fails to capture the relationship between an individual being a man and that individual being a mortal, and it quickly becomes impractical for complex worlds.

    First-Order Logic (FOL) / Predicate Logic

    First-order predicate logic (FOPL) models the world in terms of:

    • Objects: Things with individual identities (e.g., people, houses, numbers).
    • Properties: Characteristics of objects that distinguish them from other objects (e.g., red, round, prime).
    • Relations: Relationships that hold among sets of objects (e.g., brother-of, bigger-than, part-of).
    • Functions: A subset of relations where there is only one "value" for any given "input" (e.g., father-of, best-friend).

    Key components of FOL:

    • Constants: Specific objects (e.g., Marcus, Rome, 2).
    • Variables: Refer to an object without naming it (e.g., x, y).
    • Predicates (Relations): Refer to properties or relations (e.g., man(Marcus), loyalto(Marcus, Caesar)).
    • Functions: Refer to objects (e.g., father_of(John)).
    • Connectives: ∧ (and), ∨ (or), ¬ (not), → (implies), ↔ (equivalent).
    • Quantifiers:
      • Universal Quantifier (∀): "For all..." Example: ∀x P(x) means P holds for all x.   ∀x dolphin(x) → mammal(x).
      • Existential Quantifier (∃): "There exists..." Example: ∃x P(x) means P holds for some x.   ∃x mammal(x) ∧ lays_eggs(x) (platypus).
    • Well-Formed Formulas (wffs): Sentences containing no "free" variables (all variables are bound by quantifiers).
    Example: Representing Sentences in FOL
    1. Marcus was a man. → man(Marcus)
    2. Marcus was a Pompeian. → Pompeian(Marcus)
    3. All Pompeians were Romans. → ∀x: Pompeian(x) → Roman(x)
    4. Caesar was a ruler. → ruler(Caesar)
    5. All Pompeians were either loyal to Caesar or hated him. (inclusive-or) → ∀x: Pompeian(x) → loyalto(x, Caesar) ∨ hate(x, Caesar)
    6. Everyone is loyal to someone. → ∀x ∃y: loyalto(x, y)
    7. People only try to assassinate rulers they are not loyal to. → ∀x ∀y: person(x) ∧ ruler(y) ∧ try_assassinate(x, y) → ¬loyalto(x, y)
    8. Marcus tried to assassinate Caesar. → try_assassinate(Marcus, Caesar)

    Inference Techniques

    Inference is the process of deriving new knowledge (conclusions) from existing knowledge (premises).

    1. Resolution

    Resolution is a powerful inference rule used in automated theorem proving. It operates on statements in Conjunctive Normal Form (CNF), i.e., a conjunction of clauses, where each clause is a disjunction of literals.

    Propositional Resolution:

    1. Convert all propositions to clause form.
    2. Negate the proposition to be proved (P) and convert the result to clause form. Add it to the set of clauses.
    3. Repeat until a contradiction (empty clause) is found or no progress can be made:
      1. Select two clauses (parent clauses).
      2. Resolve them together: The resolvent is the disjunction of all literals of both parents, except if there is a pair of complementary literals (L and ¬L), one from each parent, these are eliminated.
      3. If the resolvent is the empty clause, a contradiction is found, and P is proved.

    Unification: In predicate logic, resolution requires unification to make literals match by finding suitable substitutions for variables.

    Example: Given (Winter ∨ Summer) and (¬Winter ∨ Cold). Resolving these yields (Summer ∨ Cold).
    2. Natural Deduction

    Natural deduction systems consist of a set of rules of inference for deriving consequences from premises in a step-by-step fashion, mimicking human logical reasoning. Proofs are constructed as trees.

    Common Rules:

    • Modus Ponens: From P and P → Q, infer Q.
    • Conjunction Introduction (∧-intro): From P and Q, infer P ∧ Q.
    • Conjunction Elimination (∧-elim): From P ∧ Q, infer P (or Q).
    • Implication Introduction (→-intro): Assume P, derive Q, then infer P → Q.
    • And rules for disjunction, negation, falsity (e.g., Reductio ad Absurdum).

    Soundness: If a system is sound, every theorem is a tautology (logically true). Completeness: If a system is complete, every tautology is a theorem.

    3. Forward Chaining (Data-Driven)

    Starts with available data (facts in the knowledge base) and uses inference rules to extract more data (derive new facts) until a goal is reached or no more rules can be fired. This strategy is often used in expert systems and production rule systems.

    Forward Chaining Example

    Image: Diagram showing facts leading to decisions (Fact 1 AND Fact 2 → Decision 1).
    4. Backward Chaining (Goal-Driven)

    Starts with a list of goals and works backward to see if there is data and rules which support any of these goals. If a rule's conclusion matches a goal, then its premises become new sub-goals. This strategy is also common in expert systems, especially for diagnostic tasks.

    Other KR Structures

    1. Semantic Networks

    A graphical KR scheme where knowledge is represented as a network of nodes (concepts, objects) and labeled arcs (relationships between nodes). Supports inheritance through ISA links. Similar to the "Inheritable Knowledge" diagram shown earlier.

    2. Frames

    Frames are data structures for representing stereotyped situations or concepts. A frame has a name and a set of slots, which can hold values (fillers). Slots can also have associated procedures (e.g., if-needed, if-added). Frames support inheritance through ISA or AKO (A Kind Of) links.

    3. Scripts

    Scripts are frame-like structures that describe stereotyped sequences of events. They include entry conditions, results, props, roles, and scenes. Useful for understanding stories or common situations.

    4. Ontologies

    An ontology is a formal, explicit specification of a shared conceptualization. It defines a common vocabulary for a domain, including types of objects/concepts, their properties, and relationships. Ontologies are crucial for knowledge sharing and reuse, especially in the Semantic Web. They often use languages like RDF, RDFS, and OWL.

    Reasoning Under Uncertainty

    Real-world knowledge is often incomplete, imprecise, or uncertain. AI systems need mechanisms to reason under these conditions.

    Monotonic vs. Non-monotonic Reasoning

    • Monotonic Reasoning: Adding new knowledge (axioms) never invalidates existing conclusions. Classical logic is monotonic.
    • Non-monotonic Reasoning: Adding new knowledge can invalidate previous conclusions. This is common in commonsense reasoning (e.g., "Birds fly." "Tweety is a bird." → "Tweety flies." Add: "Tweety is a penguin." → "Tweety does not fly."). Techniques include default logic and belief revision.
    1. Bayesian Networks (Belief Networks)

    Bayesian Networks are directed acyclic graphs (DAGs) where nodes represent random variables and edges represent probabilistic dependencies. Each node has an associated conditional probability table (CPT) that quantifies the effect of its parents. They allow for efficient probabilistic inference (e.g., calculating the probability of an event given some evidence).

    2. Fuzzy Logic

    Fuzzy logic is a form of many-valued logic that deals with reasoning that is approximate rather than fixed and exact. It uses membership functions to define the degree to which an element belongs to a fuzzy set (e.g., "temperature is warm" might have a membership value between 0 and 1). Fuzzy inference systems use fuzzy rules (IF-THEN rules with fuzzy propositions) to make decisions.

    Knowledge-Based Systems (KBS) & Expert Systems

    An expert system is a computer program that simulates the judgement and behavior of a human or an organization that has expert knowledge and experience in a particular field. Expert system = Knowledge + Problem-solving methods.

    Types of Knowledge-Based Systems

    Image: Hexagonal diagram showing Intelligent tutoring systems, Expert systems, Hypertext manipulation, Case-based systems, Rule-based systems.

    Components of an Expert System:

    • Knowledge Base: Contains domain-specific and high-quality knowledge (facts and rules).
    • Inference Engine: Applies logical rules to the knowledge base to derive new information or make decisions (uses forward/backward chaining).
    • User Interface: Allows a non-expert user to query the system and receive explanations.

    Expert System Architecture

    Image: Human Expert → Knowledge Engineer → Knowledge Base ↔ Inference Engine ↔ User Interface ↔ User.

    Capabilities:

    Advising, diagnosing, explaining, predicting, justifying conclusions.

    Limitations:

    Difficult knowledge acquisition, costly development, can be brittle (struggle with novel situations not covered by rules).
